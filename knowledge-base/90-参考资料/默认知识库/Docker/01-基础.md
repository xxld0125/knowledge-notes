### 1、docker build
**docker build** 命令用于构建 Docker 镜像。语法如下：

```plain
docker build [OPTIONS] PATH | URL | -
```

其中，**PATH** 表示上下文路径，**URL** 表示远程仓库地址，**-** 表示使用标准输入作为 Dockerfile。常用的选项包括：

+ **-t**：指定构建出的镜像的名称及标签。
+ **-f**：指定要使用的 Dockerfile 文件路径。
+ **--no-cache**：不使用缓存构建镜像。
+ **-q**：只显示构建出的镜像 ID。

示例：

```plain
docker build -t myimage:v1 .
```

### 2、docker run
**docker run** 命令用于启动一个容器。语法如下：

```plain
docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
```

其中，**IMAGE** 表示要启动的镜像名称及标签。常用的选项包括：

+ **-d**：以后台模式启动容器。
+ **-p**：指定容器端口与宿主机端口的映射关系。
+ **--name**：指定容器的名称。
+ **-v**：挂载宿主机目录到容器内部。
+ **--rm**：容器退出时自动删除容器。

示例：

```plain
docker run -d -p 8080:80 myimage:v1
```

### 3、docker ps
**docker ps** 命令用于列出正在运行的容器。语法如下：

```plain
docker ps [OPTIONS]
```

常用的选项包括：

+ **-a**：列出所有容器，包括已经退出的。
+ **-q**：只显示容器 ID。

示例：

```plain
docker ps -a
```

### 4、docker exec
**docker exec** 命令用于在运行中的容器内部执行命令。语法如下：

```plain
docker exec [OPTIONS] CONTAINER COMMAND [ARG...]
```

其中，**CONTAINER** 表示要执行命令的容器 ID 或名称。常用的选项包括：

+ **-i**：以交互模式运行命令。
+ **-t**：分配一个伪终端。

示例：

```plain
docker exec -it mycontainer sh
```

### 5、docker stop
**docker stop** 命令用于停止正在运行的容器。语法如下：

```plain
docker stop [OPTIONS] CONTAINER [CONTAINER...]
```

其中，**CONTAINER** 表示要停止的容器 ID 或名称。常用的选项包括：

+ **-t**：指定停止容器的超时时间。

示例：

```plain
docker stop mycontainer
```

### 6、docker rm
**docker rm** 命令用于删除已经停止的容器。语法如下：

```plain
docker rm [OPTIONS] CONTAINER [CONTAINER...]
```
