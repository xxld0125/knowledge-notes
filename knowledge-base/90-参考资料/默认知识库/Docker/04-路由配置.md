#### 1、问题
当前端项目使用router时,  访问对应路由的链接，会显示404。

![](https://cdn.nlark.com/yuque/0/2023/png/25743026/1683709492086-ebc1fb96-72b0-4262-a40f-ae21efa7a37a.png)



原因就是在静态资源中并没有 about 或者 about.html 该资源，因此返回 404 Not Found。而在单页应用中，/about 是由前端通过 history API 进行控制。

解决方法也很简单：在服务端将所有页面路由均指向 index.html，而单页应用再通过 history API 控制当前路由显示哪个页面。 这也是静态资源服务器的重写(Rewrite)功能。

我们在使用 nginx 镜像部署前端应用时，可通过挂载 nginx 配置解决该问题。



#### 2、nginx 的 try_files 指令
在 nginx 中，可通过 try_files 指令将所有页面导向 index.html。

```plain
location / {
    # 如果资源不存在，则回退到 index.html
    try_files  $uri $uri/ /index.html;  
}
```

此时，可解决服务器端路由问题。

除此之外，我们还可以通过 nginx 配置解决更多问题。
