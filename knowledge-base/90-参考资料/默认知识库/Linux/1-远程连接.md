### 一、ssh
#### 1、登陆服务器：ssh
ssh，`secure shell protocol`，以更加安全的方式连接远程服务器。是一种<font style="color:rgb(51, 51, 51);">专为</font>远程登录<font style="color:rgb(51, 51, 51);">会话和其他网络服务提供安全性的协议。利用 SSH 协议可以有效防止远程管理过程中的信息泄露问题。</font>

<font style="color:rgb(51, 51, 51);"></font>

把以下 IP 地址替换为你云服务器的公网地址，并提供密码即可登录。

但记住一个 IP 地址，这是一个反人性的操作，如果你有多个服务器需要管理呢？

```bash
# root: 用户名
# 47.94.168.59: 云服务器 IP 地址，需要把 IP 地址换成个人 IP 地址
$ ssh root@47.94.168.59
```



SSH协议默认端口号是TCP 22端口，TCP是`Transmission Control Protocol`的缩写，中文意思为传输控制协议，是为了在不可靠的互联网络上提供可靠的端到端字节流而专门设计的一个传输协议。



#### 2、<font style="color:rgb(44, 62, 80);">配置别名快速登录：ssh-config</font>
在本地客户端环境 (个人电脑) 上配置 ssh-config，对自己管理的服务器起别名，可以更方便地登录多台云服务器，以下是关于 ssh-config 的配置文件。

`/etc/ssh/ssh_config`

`~/.ssh/config`

登陆服务器修改配置

```bash
# 修改 ssh 配置文件 /etc/ssh/ssh_config or ~/.ssh/config
Host	tx01
    HostName 43.138.250.200
    User root

# 请用真实 IP 地址替换以下的 PUBLIC_IP
# 并记得替换 User
Host name
    HostName <PUBLIC_IP>
    User root
```

<font style="color:rgb(44, 62, 80);">配置成功之后直接 ssh 就可以直接登录</font>

```bash
xulingfeng@xulingfengdeMac-mini ~ % ssh xulf

```



#### 3、<font style="color:rgb(44, 62, 80);">免密登录：public-key 与 ssh-copy-id</font>
如何实现远程服务器的免密登录需要两个条件:

1. 两个文件: 本地环境的 `~/.ssh/id_rsa.pub` 与 远程服务器的 `~/.ssh/authorized_keys`
2. 一个动作: 把本地文件 `~/.ssh/id_rsa.pub` 中内容复制粘贴到远程服务器 `~/.ssh/authorized_keys`

> :::tips
<font style="color:rgb(153, 153, 153);">如果本地没有 </font><font style="color:rgb(71, 101, 130);">~/.ssh/id_rsa.pub</font><font style="color:rgb(153, 153, 153);"> 文件，则使用命令 </font><font style="color:rgb(71, 101, 130);">ssh keygen</font><font style="color:rgb(153, 153, 153);"> 进行生成。</font>
>
:::


**总结成一句话，把自己的公钥放在远程服务器的 **`**authorized_keys**`** 中**



#### 4、安全性：禁用密码登陆
为了更大保障服务器的安全性，这里禁止密码登录。修改云服务器的 sshd 配置文件：`/etc/ssh/sshd_config`。其中 `PasswordAuthentication` 设置为 `no`，以此来禁用密码登录。

```bash
# 编辑服务器端的 /etc/ssh/sshd_config
# 禁用密码登录

Host *
  PasswordAuthentication no
```

#### 5、保持连接，防止断掉
除此之外，还可以通过一些配置来更好地优化我们连接服务器时的体验。

我们可以通过 `man ssh config`，找到每一项的详细释义。

```bash
# 编辑 ~/.ssh/config

Host *
  ServerAliveInterval 30
  TCPKeepAlive yes
  ServerAliveCountMax 6
  Compression yes # 指定是否使用压缩, 默认为不开启
```
