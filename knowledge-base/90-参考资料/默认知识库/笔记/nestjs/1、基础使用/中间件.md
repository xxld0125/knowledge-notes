<font style="color:rgb(77, 77, 77);">中间件是在路由处理程序 </font>**之前**<font style="color:rgb(77, 77, 77);"> 调用的函数。 中间件函数可以访问请求和响应对象，以及应用程序请求响应周期中的 </font>`next()`<font style="color:rgb(77, 77, 77);">中间件函数。 </font>`next()`<font style="color:rgb(77, 77, 77);"> 中间件函数通常由名为 </font>`next`<font style="color:rgb(77, 77, 77);"> 的变量表示。</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1669199128110-e439533d-74ac-49c3-a2a3-3dd63375290b.png)

`<font style="color:rgb(77, 77, 77);">Nest</font>`<font style="color:rgb(77, 77, 77);"> 中间件实际上等价于 </font>`express`<font style="color:rgb(77, 77, 77);"> 中间件。 下面是</font>`<font style="color:rgb(77, 77, 77);">Express</font>`<font style="color:rgb(77, 77, 77);">官方文档中所述的中间件功能：</font>

<font style="color:rgb(77, 77, 77);">中间件函数可以执行以下任务:</font>

+ <font style="color:rgb(77, 77, 77);">执行任何代码。</font>
+ <font style="color:rgb(77, 77, 77);">对请求和响应对象进行更改。</font>
+ <font style="color:rgb(77, 77, 77);">结束请求-响应周期。</font>
+ <font style="color:rgb(77, 77, 77);">调用堆栈中的下一个中间件函数。</font>
+ <font style="color:rgb(77, 77, 77);">如果当前的中间件函数没有结束请求-响应周期, 它必须调用 </font>next()<font style="color:rgb(77, 77, 77);"> 将控制传递给下一个中间件函数。否则, 请求将被挂起。</font>

#### 一、依赖注入中间件
注册中间件。

```typescript
import { Injectable, NestMiddleware } from '@nestjs/common';
import { Request, Response, NextFunction } from 'express';

@Injectable()
export class LoggerMiddleware implements NestMiddleware {
  use(req: Request, res: Response, next: NextFunction) {
    console.log('middleware');
    next();
  }
}
```



使用方法 在模块里面 实现 `configure` 返回一个消费者  `consumer` 通过 `apply` 注册中间件 通过`forRoutes` 指定  `Controller` 路由。

`forRoutes`支持三种方式：

    - 指定路由（支持多个，正则）
    - 指定请求类型
    - 指定`controller`（支持多个）

```typescript
import {
  MiddlewareConsumer,
  Module,
  NestModule,
  RequestMethod,

import { UserService } from './user.service';
import { LoggerMiddleware } from '../logger/logger.middleware';

@Module({
  controllers: [UserController],
  providers: [UserService,],
})
export class UserModule implements NestModule {
  configure(consumer: MiddlewareConsumer) {
    // 一、指定路由：支持正则
    // consumer.apply(LoggerMiddleware).forRoutes('user');

    // 二、指定请求类型
    // consumer.apply(LoggerMiddleware).forRoutes({
    //   path: 'user',
    //   method: RequestMethod.POST,
    // });

  }
}

```



#### 二、全局中间件
**全局中间件只能使用函数模式, ** 案例可以做白名单拦截之类的。

```typescript
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

const whiteList = ['/list'];

function middleWareAll(req, res, next) {
  if (whiteList.includes(req.originalUrl)) {
    next();
  } else {
    res.send('非白名单路由');
  }
}

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  
  app.use(middleWareAll);

  await app.listen(3000);
}
bootstrap();

```



#### 三、接入第三方中间件
`cors`插件支持接口跨域。

`<font style="color:rgb(77, 77, 77);">npm install cors</font>`

`<font style="color:rgb(77, 77, 77);">npm install @types/cors -D</font>`

```typescript
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';
import * as cors from 'cors';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  app.use(cors());

  await app.listen(3000);
}
bootstrap();

```
