#### 三、`yarn yundoc`命令报错` Token "types.request_source.RequestSource" does not exist.`
问题代码：

```typescript
import * as types from '../grpc-code-gen/kefu/kefu-setting-provider/types';

interface GetInitDataRes {
  // 业务环节列表
  business_step: types.parameter.Value[];
  // 投诉级别列表
  complaint_level: types.parameter.Value[];
  // 问题部位列表
  problem_position: types.parameter.Value[];
  // 紧急程度列表
  task_urgency: types.parameter.Value[];
  // 请求来源列表
  request_source: types.request_source.RequestSource[];
  // 维修工期列表
  repairs_duration: types.parameter.Value[];
}
```

	

修改后代码：

```typescript
import * as types from '../grpc-code-gen/kefu/kefu-setting-provider/types';

type ParameterValueList = types.parameter.Value[];

interface GetInitDataRes {
  // 业务环节列表
  business_step: ParameterValueList;
  // 投诉级别列表
  complaint_level: ParameterValueList;
  // 问题部位列表
  problem_position: ParameterValueList;
  // 紧急程度列表
  task_urgency: ParameterValueList;
  // 请求来源列表
  request_source: types.request_source.RequestSource[];
  // 维修工期列表
  repairs_duration: ParameterValueList;
}

```



#### 四、接口报错：`error: Error: interior hyphen`
通过`grpc`调用`go`服务时，对于要求 `int32` & `int64`格式的入参，要做特殊处理，否则`yunfly`框架无法对数据进行格式转化，到时该错误抛出。

`int32`: `js`类型为`number`;

`int64`：`js`类型为`string`(先将数据转化为`number`，再转为`string`格式);
