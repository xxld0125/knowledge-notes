#### 什么是二叉树节点的深度和高度<font style="color:rgb(44, 62, 80);">。</font>
+ <font style="color:rgb(44, 62, 80);">二叉树节点的深度：指从根节点到该节点的最长简单路径边的条数。</font>
+ <font style="color:rgb(44, 62, 80);">二叉树节点的高度：指从该节点到叶子节点的最长简单路径边的条数。</font>

<font style="color:rgb(44, 62, 80);"></font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1653316835839-9e2e564a-9ee6-4f70-a2c7-16d1f9ed196b.png)



<font style="color:rgb(44, 62, 80);">因为求深度可以从上到下去查 所以需要前序遍历（中左右），而高度只能从下到上去查，所以只能后序遍历（左右中）</font>

**<font style="color:rgb(44, 62, 80);"></font>**

<font style="color:rgb(44, 62, 80);">有的同学一定疑惑，为什么</font>[09-二叉树的最大深度](https://www.yuque.com/u25370234/kb/gg332k)<font style="color:rgb(44, 62, 80);">中求的是二叉树的最大深度，也用的是后序遍历。</font>

**<font style="color:rgb(44, 62, 80);">那是因为代码的逻辑其实是求的根节点的高度，而根节点的高度就是这棵树的最大深度，所以才可以使用后序遍历。</font>**



<font style="color:rgb(44, 62, 80);">在</font>[09-二叉树的最大深度](https://www.yuque.com/u25370234/kb/gg332k)<font style="color:rgb(44, 62, 80);">中，如果真正求取二叉树的最大深度，代码应该写成如下：	（前序遍历）</font>

```javascript
function maxDepth(root) {
  let res = 0;
  if (root === null) return res;
  
  const getDepth(node, depth) {
    res = Math.max(depth, res); // 中
    
    if (node.left === null && node.right === null) return;
    
    if (node.left) { // 左
      getDepth(node.left, depth + 1);
    }
    
    if (node.right) { // 右
      getDepth(node.right, depth + 1);
    }
    return;
  }

  getDepth(root, 1);
  return res;
}
```



#### <font style="color:rgb(44, 62, 80);">思路</font>
##### 递归
题目要求的是比较高度, 必然是后序遍历<font style="color:rgb(44, 62, 80);">。</font>



递归三部曲分析:

1.明确递归函数的参数和返回值:

参数: 当前的传入节点<font style="color:rgb(44, 62, 80);">。</font>

返回值: 以当前传入节点为根节点的树的高度<font style="color:rgb(44, 62, 80);">,  返回 -1则便是左右子树的高度差值大于1, 已经不符合平衡树的规则。</font>



2.明确终止条件

递归过程中遇到空节点为终止,<font style="color:rgb(44, 62, 80);">返回0，表示当前节点为根节点的树高度为0。</font>

<font style="color:rgb(44, 62, 80);"></font>

3.明确单层递归的逻辑

<font style="color:rgb(44, 62, 80);">如何判断以当前传入节点为根节点的二叉树是否是平衡二叉树呢？当然是其左子树高度和其右子树高度的差值。</font>

<font style="color:rgb(44, 62, 80);">分别求出其左右子树的高度，然后如果差值小于等于1，则返回当前二叉树的高度，否则则返回-1，表示已经不是二叉平衡树了。</font>



```javascript
function isBalanced(root) {
  const getHeight(node) {
    if (node === null) return 0;
    
    let leftHeight = getHeight(node.left);
    if (leftHeight === -1) return -1;
    let rightHeight = getHeight(node.right);
    if (rightHeight === -1) return -1;
    return Math.abs(leftHeight - rightHeight) > 1 ? -1 : 1 + Math.max(leftHeight, rightHeight);
  }
  
  return getHeight(root) !== -1;
}
```



##### <font style="color:rgb(44, 62, 80);">迭代</font>
<font style="color:rgb(44, 62, 80);">在</font>[09-二叉树的最大深度](https://www.yuque.com/u25370234/kb/gg332k)<font style="color:rgb(44, 62, 80);">中我们可以使用层序遍历来求深度，但是就不能直接用层序遍历来求高度了，这就体现出求高度和求深度的不同。</font>

<font style="color:rgb(44, 62, 80);">本题的迭代方式可以先定义一个函数，专门用来求高度。</font>

<font style="color:rgb(44, 62, 80);">这个函数通过栈模拟的后序遍历找每一个节点的高度（其实是通过求传入节点为根节点的最大深度来求的高度）。</font>

<font style="color:rgb(44, 62, 80);">然后再用栈来模拟后序遍历，遍历每一个节点的时候，再去判断左右孩子的高度是否符合，代码如下。</font>




<font style="color:rgb(44, 62, 80);">当然此题用迭代法，其实效率很低，因为没有很好的模拟回溯的过程，所以迭代法有很多重复的计算。</font>

<font style="color:rgb(44, 62, 80);">虽然理论上所有的递归都可以用迭代来实现，但是有的场景难度可能比较大。</font>

**<font style="color:rgb(44, 62, 80);">例如：都知道回溯法其实就是递归，但是很少人用迭代的方式去实现回溯算法！</font>**

<font style="color:rgb(44, 62, 80);">因为对于回溯算法已经是非常复杂的递归了，如果在用迭代的话，就是自己给自己找麻烦，效率也并不一定高。</font>

### <font style="color:rgb(44, 62, 80);">二、总结</font>
<font style="color:rgb(44, 62, 80);">通过本题可以了解求二叉树深度 和 二叉树高度的差异，求深度适合用前序遍历，而求高度适合用后序遍历。</font>
