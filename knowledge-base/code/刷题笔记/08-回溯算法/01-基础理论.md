### 二、什么是回溯法
<font style="color:rgb(44, 62, 80);">回溯法也可以叫做回溯搜索法，它是一种搜索的方式。</font>

<font style="color:rgb(44, 62, 80);">在二叉树系列中，我们已经不止一次，提到了回溯，例如</font>[15-二叉树中递归带着回溯](https://www.yuque.com/u25370234/kb/pxesvw)<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">回溯是递归的副产品，只要有递归就会有回溯。</font>

**<font style="color:rgb(44, 62, 80);">所以以下讲解中，回溯函数也就是递归函数，指的都是一个函数</font>**<font style="color:rgb(44, 62, 80);">。</font>



### 三、回溯法的效率
<font style="color:rgb(44, 62, 80);">回溯法的性能如何呢，这里要和大家说清楚了，</font>**<font style="color:rgb(44, 62, 80);">虽然回溯法很难，很不好理解，但是回溯法并不是什么高效的算法</font>**<font style="color:rgb(44, 62, 80);">。</font>

**<font style="color:rgb(44, 62, 80);">因为回溯的本质是穷举，穷举所有可能，然后选出我们想要的答案</font>**<font style="color:rgb(44, 62, 80);">，如果想让回溯法高效一些，可以加一些剪枝的操作，但也改不了回溯法就是穷举的本质。</font>

<font style="color:rgb(44, 62, 80);">那么既然回溯法并不高效为什么还要用它呢？</font>

<font style="color:rgb(44, 62, 80);">因为没得选，一些问题能暴力搜出来就不错了，撑死了再剪枝一下，还没有更高效的解法。</font>

<font style="color:rgb(44, 62, 80);">此时大家应该好奇了，都什么问题，这么牛逼，只能暴力搜索。</font>

<font style="color:rgb(44, 62, 80);"></font>

### 四、回溯法解决的问题
<font style="color:rgb(44, 62, 80);">回溯法，一般可以解决如下几种问题：</font>

+ <font style="color:rgb(44, 62, 80);">组合问题：N个数里面按一定规则找出k个数的集合</font>
+ <font style="color:rgb(44, 62, 80);">切割问题：一个字符串按一定规则有几种切割方式</font>
+ <font style="color:rgb(44, 62, 80);">子集问题：一个N个数的集合里有多少符合条件的子集</font>
+ <font style="color:rgb(44, 62, 80);">排列问题：N个数按一定规则全排列，有几种排列方式</font>
+ <font style="color:rgb(44, 62, 80);">棋盘问题：N皇后，解数独等等</font>

<font style="color:rgb(44, 62, 80);"></font>

**<font style="color:rgb(44, 62, 80);">相信大家看着这些之后会发现，每个问题，都不简单！</font>**

<font style="color:rgb(44, 62, 80);">另外，会有一些同学可能分不清什么是组合，什么是排列？</font>

**<font style="color:rgb(44, 62, 80);">组合是不强调元素顺序的，排列是强调元素顺序</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">例如：{1, 2} 和 {2, 1} 在组合上，就是一个集合，因为不强调顺序，而要是排列的话，{1, 2} 和 {2, 1} 就是两个集合了。</font>

<font style="color:rgb(44, 62, 80);">记住组合无序，排列有序，就可以了。</font>



### 五、如何理解回溯法
**<font style="color:rgb(44, 62, 80);">回溯法解决的问题都可以抽象为树形结构</font>**<font style="color:rgb(44, 62, 80);">，是的，我指的是所有回溯法的问题都可以抽象为树形结构！</font>

<font style="color:rgb(44, 62, 80);">因为回溯法解决的都是在集合中递归查找子集，</font>**<font style="color:rgb(44, 62, 80);">集合的大小就构成了树的宽度，递归的深度，都构成的树的深度</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">递归就要有终止条件，所以必然是一棵高度有限的树（N叉树）。</font>

<font style="color:rgb(44, 62, 80);">这块可能初学者还不太理解，后面的回溯算法解决的所有题目中，我都会强调这一点并画图举相应的例子，现在有一个印象就行。</font>

<font style="color:rgb(44, 62, 80);"></font>

### 六、回溯法模版
在二叉树的递归中，我们介绍了递归三部曲，这里再介绍回溯三部曲。

+ 回溯函数模板返回值以及参数

在回溯算法中，我的习惯是函数起名字为backtracking，这个起名大家随意。

回溯算法中函数返回值一般为void。

再来看一下参数，因为回溯算法需要的参数可不像二叉树递归的时候那么容易一次性确定下来，所以一般是先写逻辑，然后需要什么参数，就填什么参数。

但后面的回溯题目的讲解中，为了方便大家理解，我在一开始就帮大家把参数确定下来。

回溯函数伪代码如下：

```javascript
function backtracking(参数) {}
```

<font style="color:rgb(44, 62, 80);"></font>

+ <font style="color:rgb(44, 62, 80);">回溯函数终止条件</font>

<font style="color:rgb(44, 62, 80);">既然是树形结构，那么我们在讲解二叉树递归的时候，就知道遍历树形结构一定要有终止条件。</font>

<font style="color:rgb(44, 62, 80);">所以回溯也有要终止条件。</font>

<font style="color:rgb(44, 62, 80);">什么时候达到了终止条件，树中就可以看出，一般来说搜到叶子节点了，也就找到了满足条件的一条答案，把这个答案存放起来，并结束本层递归。</font>

<font style="color:rgb(44, 62, 80);">所以回溯函数终止条件伪代码如下：</font>

```javascript
if (终止条件} {
    存放结果;
    return;
}
```

<font style="color:rgb(44, 62, 80);background-color:rgb(40, 44, 52);"></font>

+ <font style="color:rgb(44, 62, 80);">回溯搜索的遍历过程</font>

<font style="color:rgb(44, 62, 80);">在上面我们提到了，回溯法一般是在集合中递归搜索，集合的大小构成了树的宽度，递归的深度构成的树的深度。</font>

<font style="color:rgb(44, 62, 80);">如图：</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1658125494145-96ff45b3-49f2-47db-9f96-36e14a52495d.png)

<font style="color:rgb(44, 62, 80);">注意图中，我特意举例集合大小和孩子的数量是相等的！</font>

<font style="color:rgb(44, 62, 80);">回溯函数遍历过程伪代码如下：</font>

```javascript
for (选择：本层集合中元素（树中节点孩子的数量就是集合的大小）) {
    处理节点;
    backtracking(路径，选择列表); // 递归
    回溯，撤销处理结果
}
```

<font style="color:rgb(44, 62, 80);">for循环就是遍历集合区间，可以理解一个节点有多少个孩子，这个for循环就执行多少次。</font>

<font style="color:rgb(44, 62, 80);">backtracking这里自己调用自己，实现递归。</font>

<font style="color:rgb(44, 62, 80);">大家可以从图中看出</font>**<font style="color:rgb(44, 62, 80);">for循环可以理解是横向遍历，backtracking（递归）就是纵向遍历</font>**<font style="color:rgb(44, 62, 80);">，这样就把这棵树全遍历完了，一般来说，搜索叶子节点就是找的其中一个结果了。</font>

<font style="color:rgb(44, 62, 80);">分析完过程，回溯算法模板框架如下：</font>

```javascript
void backtracking(参数) {
    if (终止条件) {
        存放结果;
        return;
    }

    for (选择：本层集合中元素（树中节点孩子的数量就是集合的大小）) {
        处理节点;
        backtracking(路径，选择列表); // 递归
        回溯，撤销处理结果
    }
}
```

**<font style="color:rgb(44, 62, 80);">这份模板很重要，后面做回溯法的题目都靠它了！</font>**

<font style="color:rgb(44, 62, 80);">如果从来没有学过回溯算法的录友们，看到这里会有点懵，后面开始讲解具体题目的时候就会好一些了，已经做过回溯法题目的录友，看到这里应该会感同身受了。</font>
