如果题目**限制了数值范围**(类似数值都是字母的场景), 可以优先考虑使用数组做哈希的题目,因为数组占用的空间和速度都快于`set` ;



如果<font style="color:rgb(44, 62, 80);">没有</font>**<font style="color:rgb(44, 62, 80);">限制数值范围</font>**<font style="color:rgb(44, 62, 80);">,就无法使用数组来做哈希表了, 且当哈希值比较少、特别分散、跨度非常大，使用数组就造成空间的极大浪费</font>**<font style="color:rgb(44, 62, 80);">。</font>**

<font style="color:rgb(44, 62, 80);">且题目特意说明：</font>**<font style="color:rgb(44, 62, 80);">输出结果中的每个元素一定是唯一的，也就是说输出的结果的去重的， 同时可以不考虑输出结果的顺序</font>**<font style="color:rgb(44, 62, 80);">, 故该题使用 </font>`<font style="color:rgb(44, 62, 80);">set</font>`<font style="color:rgb(44, 62, 80);"> 哈希数据结构</font>**<font style="color:rgb(44, 62, 80);">。</font>**

**<font style="color:rgb(44, 62, 80);"></font>**

**代码:**

```javascript
/**
 * @param {number[]} nums1
 * @param {number[]} nums2
 * @return {number[]}
 */
var intersection = function(nums1, nums2) {
    if (nums1.length < nums2.length) {
        [nums1, nums2] = [nums2, nums1];
    }

    const set1 = new Set(nums1);
    const resSet = new Set;
    
    // for(const n of nums2) {
    //     nums1Set.has(n) && resSet.add(n);
    // }
    // 循环 比 迭代器快
  
    for (let i = 0;i < nums2.length;i++) {
        set1.has(nums2[i]) && resSet.add(nums2[i]); 
    }

    return Array.from(resSet);
};
```



### 二、[350. 两个数组的交集 II](https://leetcode.cn/problems/intersection-of-two-arrays-ii/)
**进阶问题**：



如果给定的数组已经排好序呢？你将如何优化你的算法？

如果 nums1 的大小比 nums2 小，哪种方法更优？

如果 nums2 的元素存储在磁盘上，内存是有限的，并且你不能一次加载所有的元素到内存中，你该怎么办？



#### 解法一: 哈希表


```javascript
/**
 * @param {number[]} nums1
 * @param {number[]} nums2
 * @return {number[]}
 */
var intersect = function(nums1, nums2) {
    const map1 = new Map;

    for (let i = 0;i < nums1.length;i++) {
        map1.set(nums1[i], (map1.get(nums1[i]) || 0) + 1);
    }

    const res = [];

    for (let i = 0;i < nums2.length;i++) {
        if (map1.get(nums2[i]) > 0) {
            res.push(nums2[i]);
            map1.set(nums2[i], map1.get(nums2[i]) - 1);
        }
    } 

    return res;
};
```



#### 解法二:排序 + 双指针


```javascript
/**
 * @param {number[]} nums1
 * @param {number[]} nums2
 * @return {number[]}
 */
var intersect = function(nums1, nums2) {
    nums1.sort((a, b) => a - b);

    nums2.sort((a, b) => a - b);

    let l1 = 0,
        l2 = 0;

    const res = [];

    while (l1 < nums1.length && l2 < nums2.length) {
        if (nums1[l1] === nums2[l2]) {
            res.push(nums1[l1]);
            l1++;
            l2++;
        } else if (nums1[l1] < nums2[l2]) {
            l1++;
        } else {
            l2++;
        }
    }
    return res;
};
```

####   
进阶问题:
如果 `nums2`的元素存储在磁盘上，磁盘内存是有限的，并且你不能一次加载所有的元素到内存中。那么就无法高效地对 `nums2`进行排序，因此推荐使用方法一而不是方法二。在方法一中，`nums2`只关系到查询操作，因此每次读取`nums2`中的一部分数据，并进行处理即可。
