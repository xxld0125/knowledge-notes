### 一、例题-[150. 逆波兰表达式求值](https://leetcode.cn/problems/evaluate-reverse-polish-notation/)
<font style="color:rgb(44, 62, 80);">根据 逆波兰表示法，求表达式的值。</font>

<font style="color:rgb(44, 62, 80);">有效的运算符包括 + ,  - ,  * ,  / 。每个运算对象可以是整数，也可以是另一个逆波兰表达式。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">说明：</font>

<font style="color:rgb(44, 62, 80);">整数除法只保留整数部分。 给定逆波兰表达式总是有效的。换句话说，表达式总会得出有效数值且不存在除数为 0 的情况。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">示例 1：</font>

+ <font style="color:rgb(44, 62, 80);">输入: ["2", "1", "+", "3", " * "]</font>
+ <font style="color:rgb(44, 62, 80);">输出: 9</font>
+ <font style="color:rgb(44, 62, 80);">解释: 该算式转化为常见的中缀算术表达式为：((2 + 1) * 3) = 9</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">示例 2：</font>

+ <font style="color:rgb(44, 62, 80);">输入: ["4", "13", "5", "/", "+"]</font>
+ <font style="color:rgb(44, 62, 80);">输出: 6</font>
+ <font style="color:rgb(44, 62, 80);">解释: 该算式转化为常见的中缀算术表达式为：(4 + (13 / 5)) = 6</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">示例 3：</font>

+ <font style="color:rgb(44, 62, 80);">输入: ["10", "6", "9", "3", "+", "-11", " * ", "/", " * ", "17", "+", "5", "+"]</font>
+ <font style="color:rgb(44, 62, 80);">输出: 22</font>
+ <font style="color:rgb(44, 62, 80);">解释:该算式转化为常见的中缀算术表达式为：</font>

```javascript
((10 * (6 / ((9 + 3) * -11))) + 17) + 5       
= ((10 * (6 / (12 * -11))) + 17) + 5       
= ((10 * (6 / -132)) + 17) + 5     
= ((10 * 0) + 17) + 5     
= (0 + 17) + 5    
= 17 + 5    
= 22   
```

<font style="color:rgb(44, 62, 80);">逆波兰表达式：是一种后缀表达式，所谓后缀就是指算符写在后面。</font>

<font style="color:rgb(44, 62, 80);">平常使用的算式则是一种中缀表达式，如 ( 1 + 2 ) * ( 3 + 4 ) 。</font>

<font style="color:rgb(44, 62, 80);">该算式的逆波兰表达式写法为 ( ( 1 2 + ) ( 3 4 + ) * ) 。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">逆波兰表达式主要有以下两个优点：</font>

+ <font style="color:rgb(44, 62, 80);">去掉括号后表达式无歧义，上式即便写成 1 2 + 3 4 + * 也可以依据次序计算出正确结果。</font>
+ <font style="color:rgb(44, 62, 80);">适合用栈操作运算：遇到数字则入栈；遇到算符则取出栈顶两个数字进行计算，并将结果压入栈中</font>[#](https://www.programmercarl.com/0150.%E9%80%86%E6%B3%A2%E5%85%B0%E8%A1%A8%E8%BE%BE%E5%BC%8F%E6%B1%82%E5%80%BC.html#%E6%80%9D%E8%B7%AF)  




#### 思路:
**<font style="color:rgb(44, 62, 80);">栈与递归之间在某种程度上是可以转换的！</font>**

**<font style="color:rgb(44, 62, 80);"></font>**

**<font style="color:rgb(44, 62, 80);">其实逆波兰表达式相当于是二叉树中的后序遍历</font>**<font style="color:rgb(44, 62, 80);">。 大家可以把运算符作为中间节点，按照后序遍历的规则画出一个二叉树。</font>

<font style="color:rgb(44, 62, 80);">但我们没有必要从二叉树的角度去解决这个问题，只要知道逆波兰表达式是用后续遍历的方式把二叉树序列化了，就可以了。</font>

**<font style="color:rgb(44, 62, 80);"></font>**

<font style="color:rgb(44, 62, 80);">在进一步看，本题中每一个子表达式要得出一个结果，然后拿这个结果再进行运算，那么</font>**<font style="color:rgb(44, 62, 80);">这岂不就是一个相邻字符串消除的过程，和</font>**[04-删除字符串中的所有相邻重复项](https://www.yuque.com/u25370234/kb/eq1g6p)**<font style="color:rgb(44, 62, 80);">中的对对碰游戏是不是就非常像了。</font>**

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1652707319851-342a1114-ef27-4970-9c6c-43f69cb5c952.png)



<font style="color:rgb(44, 62, 80);">整体过程</font>**<font style="color:rgb(44, 62, 80);">和</font>**[04-删除字符串中的所有相邻重复项](https://www.yuque.com/u25370234/kb/eq1g6p)<font style="color:rgb(44, 62, 80);">差不错的，只不过本题不要相邻元素做消除了，而是做运算！</font>

<font style="color:rgb(44, 62, 80);"></font>

```javascript
/**
 * @param {string[]} tokens
 * @return {number}
 */
var evalRPN = function(tokens) {
    const s = new Map([
        ["+", (a, b) => a * 1  + b * 1],
        ["-", (a, b) => b - a],
        ["*", (a, b) => b * a],
        ["/", (a, b) => (b / a) | 0]
    ]);
    const stack = [];
    for (const i of tokens) {
        if(!s.has(i)) {
            stack.push(i);
            continue;
        }
        stack.push(s.get(i)(stack.pop(),stack.pop()))
    }
    return stack.pop();
};
```
