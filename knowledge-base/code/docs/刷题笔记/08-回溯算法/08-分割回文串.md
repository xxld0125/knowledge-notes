### 一、[131. 分割回文串](https://leetcode.cn/problems/palindrome-partitioning/)
<font style="color:rgb(44, 62, 80);">给定一个字符串 s，将 s 分割成一些子串，使每个子串都是回文串。</font>

<font style="color:rgb(44, 62, 80);">返回 s 所有可能的分割方案。</font>

<font style="color:rgb(44, 62, 80);">示例: 输入: "aab" 输出: [ ["aa","b"], ["a","a","b"] ]</font>

<font style="color:rgb(44, 62, 80);"></font>

#### <font style="color:rgb(44, 62, 80);">思路</font>
<font style="color:rgb(44, 62, 80);">本题这涉及到两个关键问题：</font>

    1. <font style="color:rgb(44, 62, 80);">切割问题，有不同的切割方式</font>
    2. <font style="color:rgb(44, 62, 80);">判断回文</font>

<font style="color:rgb(44, 62, 80);">这种题目，想用for循环暴力解法，可能都不那么容易写出来，所以要换一种暴力的方式，就是回溯。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">我们来分析一下切割，</font>**<font style="color:rgb(44, 62, 80);">其实切割问题类似组合问题</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">切割问题，也可以抽象为一棵树形结构，如图：</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1660198963055-fe5910c7-c4d5-4cf9-b71b-5772fdda2e07.png)

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">递归用来纵向遍历，for循环用来横向遍历，切割线（就是图中的红线）切割到字符串的结尾位置，说明找到了一个切割方法。</font>

<font style="color:rgb(44, 62, 80);">此时可以发现，切割问题的回溯搜索的过程和组合问题的回溯搜索的过程是差不多的。</font>

<font style="color:rgb(44, 62, 80);"></font>

#### 回溯三部曲
##### <font style="color:rgb(44, 62, 80);">递归函数参数</font>
<font style="color:rgb(44, 62, 80);">本题递归函数参数需要startIndex，因为切割过的地方，不能重复切割，和组合问题也是保持一致的。</font>

```javascript
const result = [], path = [];

const backtracking = function(startIndex) {};
```



##### 递归终止条件
![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1660797057915-59054ca2-5028-42f7-8906-9aa31482e295.png)



<font style="color:rgb(44, 62, 80);">从树形结构的图中可以看出：切割线切到了字符串最后面，说明找到了一种切割方法，此时就是本层递归的终止终止条件。</font>

**<font style="color:rgb(44, 62, 80);">那么在代码里什么是切割线呢？</font>**

<font style="color:rgb(44, 62, 80);">在处理组合问题的时候，递归参数需要传入startIndex，表示下一轮递归遍历的起始位置，		这个startIndex就是切割线。</font>

<font style="color:rgb(44, 62, 80);">所以终止条件代码如下：</font>

```javascript
const backtracking = function(startIndex) {
  // 如果起始位置已经大于s的大小，说明已经找到了一组分割方案了
  if (startIndex > s.length) {
    result.push(path);
    return;
  }
};

```



##### 单层搜索的逻辑
**<font style="color:rgb(44, 62, 80);">来看看在递归循环，中如何截取子串呢？</font>**

<font style="color:rgb(44, 62, 80);">在</font>`<font style="color:rgb(71, 101, 130);">for (let i = startIndex; i < length; i++)</font>`<font style="color:rgb(44, 62, 80);">循环中，我们 定义了起始位置</font>`<font style="color:rgb(44, 62, 80);">startIndex</font>`<font style="color:rgb(44, 62, 80);">，那么</font>`<font style="color:rgb(44, 62, 80);">[startIndex, i] </font>`<font style="color:rgb(44, 62, 80);">就是要截取的子串。</font>

<font style="color:rgb(44, 62, 80);">首先判断这个子串是不是回文，如果是回文，就加入在</font>`<font style="color:rgb(71, 101, 130);">path</font>`<font style="color:rgb(44, 62, 80);">中，</font>`<font style="color:rgb(44, 62, 80);">path</font>`<font style="color:rgb(44, 62, 80);">用来记录切割过的回文子串。</font>

<font style="color:rgb(44, 62, 80);">代码如下：</font>

```javascript
for (let i = startIndex;i < s.length;i++) {
  if (!isPalindrome(s, startIndex, i)) { // 判断是否为回文子串, 如果不是则直接跳过
    continue;
  }

  const str = s.slice(startIndex, i + 1);
  path.push(str);
  backtracking(i + 1); // 寻找i+1为起始位置的子串
  path.pop(); // 回溯过程，弹出本次已经填在的子串
}
```



#### 完整代码
```javascript
var partition = function(s) {
    const res = [], path = [];

    const isPalindrome = function(s, l, r) {
        for (let i = l, j = r; i < j;i++, j--) {
            if (s[i] !== s[j]) {
                return false;
            }
        }
        return true;
    }

    const backtracking = function(startIndex) {
        if (startIndex >= s.length) {
            res.push(Array.from(path));
            return;
        }

        for (let i = startIndex;i < s.length;i++) {
            if (!isPalindrome(s, startIndex, i)) {
                continue;
            }

            const str = s.slice(startIndex, i + 1);
            path.push(str);
            backtracking(i + 1);
            path.pop();
        }
    }
    backtracking(0);
    return res;
};
```
