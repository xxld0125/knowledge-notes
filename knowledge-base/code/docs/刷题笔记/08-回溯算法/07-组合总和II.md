### 一、[40. 组合总和 II](https://leetcode.cn/problems/combination-sum-ii/)
<font style="color:rgb(44, 62, 80);">给定一个数组 candidates 和一个目标数 target ，找出 candidates 中所有可以使数字和为 target 的组合。</font>

<font style="color:rgb(44, 62, 80);">candidates 中的每个数字在每个组合中只能使用一次。</font>

<font style="color:rgb(44, 62, 80);">说明： 所有数字（包括目标数）都是正整数。 解集不能包含重复的组合。</font>

<font style="color:rgb(44, 62, 80);">示例 1: 输入: candidates = [10,1,2,7,6,1,5], target = 8, 所求解集为: [ [1, 7], [1, 2, 5], [2, 6], [1, 1, 6] ]</font>

<font style="color:rgb(44, 62, 80);">示例 2: 输入: candidates = [2,5,2,1,2], target = 5, 所求解集为: [   [1,2,2],   [5] ]</font>

<font style="color:rgb(44, 62, 80);"></font>

#### 思路
<font style="color:rgb(44, 62, 80);">这道题目和</font>[06-组合总和](https://www.yuque.com/u25370234/kb/neqtwa)<font style="color:rgb(44, 62, 80);">如下区别：</font>

1. <font style="color:rgb(44, 62, 80);">本题candidates 中的每个数字在每个组合中只能使用一次。</font>
2. <font style="color:rgb(44, 62, 80);">本题数组candidates的元素是有重复的，而</font>[06-组合总和](https://www.yuque.com/u25370234/kb/neqtwa)<font style="color:rgb(44, 62, 80);">是无重复元素的数组candidates</font>

<font style="color:rgb(44, 62, 80);">最后本题和</font>[06-组合总和](https://www.yuque.com/u25370234/kb/neqtwa)<font style="color:rgb(44, 62, 80);">要求一样，解集不能包含重复的组合。</font>

<font style="color:rgb(44, 62, 80);"></font>

**<font style="color:rgb(44, 62, 80);">本题的难点在于区别2中：集合（数组candidates）有重复元素，但还不能有重复的组合</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">一些同学可能想了：我把所有组合求出来，再用set或者map去重，这么做很容易超时！</font>

<font style="color:rgb(44, 62, 80);">所以要在搜索的过程中就去掉重复组合。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">这个去重为什么很难理解呢，</font>**<font style="color:rgb(44, 62, 80);">所谓去重，其实就是使用过的元素不能重复选取。</font>**<font style="color:rgb(44, 62, 80);"> </font><font style="color:rgb(44, 62, 80);">这么一说好像很简单！</font>

<font style="color:rgb(44, 62, 80);">都知道组合问题可以抽象为树形结构，那么“使用过”在这个树形结构上是有两个维度的，一个维度是同一树枝上使用过，一个维度是同一树层上使用过。</font>**<font style="color:rgb(44, 62, 80);">没有理解这两个层面上的“使用过” 是造成大家没有彻底理解去重的根本原因。</font>**

**<font style="color:rgb(44, 62, 80);"></font>**

<font style="color:rgb(44, 62, 80);">那么问题来了，我们是要同一树层上使用过，还是同一树枝上使用过呢？</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">回看一下题目，元素在同一个组合内是可以重复的，怎么重复都没事，但两个组合不能相同。</font>

**<font style="color:rgb(44, 62, 80);">所以我们要去重的是同一树层上的“使用过”，同一树枝上的都是一个组合里的元素，不用去重</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">为了理解去重我们来举一个例子，candidates = [1, 1, 2], target = 3，（方便起见candidates已经排序了）</font>

**<font style="color:rgb(44, 62, 80);">强调一下，树层去重的话，需要对数组排序！</font>**

**<font style="color:rgb(44, 62, 80);"></font>**

<font style="color:rgb(44, 62, 80);">选择过程树形结构如图所示：</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1660125902431-52b12c4b-2645-4fbd-99e8-6ae549404a32.png)  
	

`<font style="color:rgb(44, 62, 80);">candidates[i] == candidates[i - 1]</font>`<font style="color:rgb(44, 62, 80);">且</font>`<font style="color:rgb(44, 62, 80);">i > startIndex</font>`<font style="color:rgb(44, 62, 80);">的情况下，说明同一数层有两个重复元素,不可以重复取数。</font>



#### 回溯三部曲
##### 递归函数参数


```javascript
const backtracking = function(sum, startIndex) {}
```



##### 递归终止条件


```javascript
if (sum === target) {
  res.push([...path]);
  return;
}
```



##### 单层搜索逻辑
<font style="color:rgb(44, 62, 80);">这里与</font>[06-组合总和](https://www.yuque.com/u25370234/kb/neqtwa)<font style="color:rgb(44, 62, 80);">最大的不同就是要去重了。</font>

<font style="color:rgb(44, 62, 80);">前面我们提到：要去重的是“同一树层上的使用过”，如果判断同一树层上元素（相同的元素）是否使用过了呢。</font>

**<font style="color:rgb(44, 62, 80);">如果</font>**`<font style="color:rgb(44, 62, 80);">candidates[i] == candidates[i - 1]</font>`<font style="color:rgb(44, 62, 80);">且</font>`<font style="color:rgb(44, 62, 80);">i > startIndex</font>`**<font style="color:rgb(44, 62, 80);">，就说明：前一个树枝，使用了candidates[i - 1]，也就是说同一树层使用过candidates[i - 1]</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">此时for循环里就应该做continue的操作。</font>

```javascript
for (let i = startIndex;i < len;i++) {
  const cur = candidates[i];

  // 若当前元素和前一个元素相等
  // 则本次循环结束，防止出现重复组合
  if (i > startIndex && cur === candidates[i - 1]) {
    continue;
  }

  // 如果当前元素值大于目标值-总和的值
  // 由于数组已排序，那么该元素之后的元素必定不满足条件
  // 直接终止当前层的递归
  if (cur > target - sum) break;

  path.push(cur);
  sum += cur;

  backtracking(sum, i + 1);

  path.pop();
  sum -= cur;
}
```



#### 完整代码
```javascript
var combinationSum2 = function(candidates, target) {
  const res = [],
        path = [],
        len = candidates.length;
 
  candidates.sort((a, b) => a - b);
  
  const backtracking = function(sum, startIndex) {
    if (sum === target) {
      res.push([...path]);
      return;
    }
    
    for (let i = startIndex;i < len;i++) {
      const cur = candidates[i];
      
      // 若当前元素和前一个元素相等
      // 则本次循环结束，防止出现重复组合
      if (i > startIndex && cur === candidates[i - 1]) {
        continue;
      }
      
      // 如果当前元素值大于目标值-总和的值
      // 由于数组已排序，那么该元素之后的元素必定不满足条件
      // 直接终止当前层的递归
      if (cur > target - sum) break;
      
      path.push(cur);
      sum += cur;

      backtracking(sum, i + 1);
      
      path.pop();
      sum -= cur;
    }
  }
  
  backtracking(0, 0);
  return res;
}
```



#### 总结：
<font style="color:rgb(44, 62, 80);">本题同样是求组合总和，但就是因为其数组</font>`<font style="color:rgb(44, 62, 80);">candidates</font>`<font style="color:rgb(44, 62, 80);">有重复元素，而要求不能有重复的组合，所以相对于</font>[06-组合总和](https://www.yuque.com/u25370234/kb/neqtwa)<font style="color:rgb(44, 62, 80);">难度提升了不少。</font>
