### 一、[77. 组合](https://leetcode.cn/problems/combinations/)
<font style="color:rgb(44, 62, 80);">给定两个整数 n 和 k，返回 1 ... n 中所有可能的 k 个数的组合。</font>

<font style="color:rgb(44, 62, 80);">示例:  
</font><font style="color:rgb(44, 62, 80);">	输入: n = 4, k = 2  
</font><font style="color:rgb(44, 62, 80);">	输出:  
</font><font style="color:rgb(44, 62, 80);">[  
</font><font style="color:rgb(44, 62, 80);">[2,4],  
</font><font style="color:rgb(44, 62, 80);">[3,4],  
</font><font style="color:rgb(44, 62, 80);">[2,3],  
</font><font style="color:rgb(44, 62, 80);">[1,2],  
</font><font style="color:rgb(44, 62, 80);">[1,3],  
</font><font style="color:rgb(44, 62, 80);">[1,4],  
</font><font style="color:rgb(44, 62, 80);">]</font>

<font style="color:rgb(44, 62, 80);"></font>

#### 思路
<font style="color:rgb(44, 62, 80);">本题这是回溯法的经典题目。</font>

<font style="color:rgb(44, 62, 80);">直接的解法当然是使用for循环，例如示例中k为2，很容易想到 用两个for循环，这样就可以输出 和示例中一样的结果。</font>

```javascript
for (let i = 1;i <= n;i++) {
  for (let j = i + 1;j <= n;j++) {
    res.push([i, j]);
  }
}
```

	

<font style="color:rgb(44, 62, 80);">输入：n = 100, k = 3 那么就三层for循环，代码如下：</font>

```javascript

for (let i = 1;i <= n;i++) {
  for (let j = i + 1;j <= n;j++) {
    for (let u = j + 1;u <= n;u++) {
      res.push([i, j, u]);  
    }
  }
}
```

	**<font style="color:rgb(44, 62, 80);">如果n为100，k为50呢，那就50层for循环，是不是开始窒息</font>**<font style="color:rgb(44, 62, 80);">。</font>

**<font style="color:rgb(44, 62, 80);">此时就会发现虽然想暴力搜索，但是用for循环嵌套连暴力都写不出来！</font>**

<font style="color:rgb(44, 62, 80);">咋整？</font>

<font style="color:rgb(44, 62, 80);">回溯搜索法来了，虽然回溯法也是暴力，但至少能写出来，不像for循环嵌套k层让人绝望。</font>

<font style="color:rgb(44, 62, 80);">那么回溯法怎么暴力搜呢？</font>

<font style="color:rgb(44, 62, 80);">上面我们说了</font>**<font style="color:rgb(44, 62, 80);">要解决 n为100，k为50的情况，暴力写法需要嵌套50层for循环，那么回溯法就用递归来解决嵌套层数的问题</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">递归来做层叠嵌套（可以理解是开k层for循环），</font>**<font style="color:rgb(44, 62, 80);">每一次的递归中嵌套一个for循环，那么递归就可以用于解决多层嵌套循环的问题了</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">此时递归的层数大家应该知道了，例如：n为100，k为50的情况下，就是递归50层。</font>

<font style="color:rgb(44, 62, 80);">一些同学本来对递归就懵，回溯法中递归还要嵌套for循环，可能就直接晕倒了！</font>

<font style="color:rgb(44, 62, 80);">如果脑洞模拟回溯搜索的过程，绝对可以让人窒息，所以需要抽象图形结构来进一步理解。</font>

<font style="color:rgb(44, 62, 80);"></font>

**<font style="color:rgb(44, 62, 80);">我们在</font>**[01-基础理论](https://www.yuque.com/u25370234/kb/pcp3d4)**<font style="color:rgb(44, 62, 80);">中说道回溯法解决的问题都可以抽象为树形结构（N叉树），用树形结构来理解回溯就容易多了</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">那么我把组合问题抽象为如下树形结构：</font>

![画板](https://cdn.nlark.com/yuque/0/2022/jpeg/25743026/1658379447220-258d31b0-4e9b-4910-b5bb-93f2bbc17452.jpeg)

	

<font style="color:rgb(44, 62, 80);">可以看出这个棵树，一开始集合是 1，2，3，4， 从左向右取数，取过的数，不在重复取。</font>

<font style="color:rgb(44, 62, 80);">第一次取1，集合变为2，3，4 ，因为k为2，我们只需要再取一个数就可以了，分别取2，3，4，得到集合[1,2] [1,3] [1,4]，以此类推。</font>

**<font style="color:rgb(44, 62, 80);">每次从集合中选取元素，可选择的范围随着选择的进行而收缩，调整可选择的范围</font>**<font style="color:rgb(44, 62, 80);">。</font>

**<font style="color:rgb(44, 62, 80);">图中可以发现n相当于树的宽度，k相当于树的深度</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">那么如何在这个树上遍历，然后收集到我们要的结果集呢？</font>

**<font style="color:rgb(44, 62, 80);">图中每次搜索到了叶子节点，我们就找到了一个结果</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">相当于只需要把达到叶子节点的结果收集起来，就可以求得 n个数中k个数的组合集合。</font>

<font style="color:rgb(44, 62, 80);"></font>

#### <font style="color:rgb(44, 62, 80);">回溯法三部曲</font>
+ 递归函数的返回值和参数

<font style="color:rgb(44, 62, 80);">在这里要定义两个全局变量，一个用来存放符合条件单一结果，一个用来存放符合条件结果的集合。</font>

```javascript
const result = []; // 存放符合条件结果的结合
const path = []; // 用来存放符合条件结果
```

<font style="color:rgb(44, 62, 80);">其实不定义这两个全局遍历也是可以的，把这两个变量放进递归函数的参数里，但函数里参数太多影响可读性，所以我定义全局变量了。</font>

<font style="color:rgb(44, 62, 80);">函数里一定有两个参数，既然是集合n里面取k的数，那么n和k是两个参数。</font>

<font style="color:rgb(44, 62, 80);">然后还需要一个参数，startIndex，这个参数用来记录本层递归的中，集合从哪里开始遍历（集合就是[1,...,n] ）。</font>

<font style="color:rgb(44, 62, 80);">为什么要有这个startIndex呢？</font>

**<font style="color:rgb(44, 62, 80);">每次从集合中选取元素，可选择的范围随着选择的进行而收缩，调整可选择的范围，就是要靠startIndex</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">从下图中红线部分可以看出，在集合[1,2,3,4]取1之后，下一层递归，就要在[2,3,4]中取数了，那么下一层递归如何知道从[2,3,4]中取数呢，靠的就是startIndex。</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1658379942699-c6bd1427-64b2-43bc-924e-0f5c4707f416.png)

<font style="color:rgb(44, 62, 80);"></font>

```javascript
const result = []; // 存放符合条件结果的结合
const path = []; // 用来存放符合条件结果

function backtracking(n, k, startIndex) {}
```



+ 回溯函数终止条件

<font style="color:rgb(44, 62, 80);">什么时候到达所谓的叶子节点了呢？</font>

<font style="color:rgb(44, 62, 80);">path这个数组的大小如果达到k，说明我们找到了一个子集大小为k的组合了，在图中path存的就是根节点到叶子节点的路径。</font>

<font style="color:rgb(44, 62, 80);">如图红色部分：</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1658380026279-3e54f459-e4c8-4e50-9600-d8be87a22c21.png)

<font style="color:rgb(44, 62, 80);">此时用result二维数组，把path保存起来，并终止本层递归。</font>

```javascript
if (path.length === k) {
  result.push(path);
  return;
}
```



+ 单层搜索条件

<font style="color:rgb(44, 62, 80);">回溯法的搜索过程就是一个树型结构的遍历过程，在如下图中，可以看出for循环用来横向遍历，递归的过程是纵向遍历。</font>

![画板](https://cdn.nlark.com/yuque/0/2022/jpeg/25743026/1658380389344-63af083e-218b-4a19-9c88-953c4d80b444.jpeg)

<font style="color:rgb(44, 62, 80);">如此我们才遍历完图中的这棵树。</font>

<font style="color:rgb(44, 62, 80);">for循环每次从startIndex开始遍历，然后用path保存取到的节点i。</font>

```javascript
for (let i = startIndex;i <= n;i++) { // 控制树的横向遍历
  path.push(i); // 处理节点 
  backtracking(n, k, i + 1); // 递归：控制树的纵向遍历，注意下一层搜索要从i+1开始
  path.pop(); // 回溯，撤销处理的节点
}
```

<font style="color:rgb(44, 62, 80);">可以看出backtracking（递归函数）通过不断调用自己一直往深处遍历，总会遇到叶子节点，遇到了叶子节点就要返回。</font>

<font style="color:rgb(44, 62, 80);">backtracking的下面部分就是回溯的操作了，撤销本次处理的结果。</font>

<font style="color:rgb(44, 62, 80);"></font>

组合问题完整代码

```javascript
 function combine(n, k) {
   const result = [];
   const path = [];

   const backtracking = function (n, k, startIndex) {
        if (path.length === k) {
            result.push([...path]);
            return;
        }

        for (let i = startIndex;i <= n;i++) {
            path.push(i); // 处理节点
            backtracking(n, k, i + 1); // 递归
            path.pop(); // 回溯, 撤销处理的节点
        }
   }

   backtracking(n, k, 1);
   return result;
 }
```

###   
三、剪枝优化
<font style="color:rgb(44, 62, 80);">我们说过，回溯法虽然是暴力搜索，但也有时候可以有点剪枝优化一下的。</font>

<font style="color:rgb(44, 62, 80);">在遍历的过程中有如下代码：</font>

<font style="color:rgb(44, 62, 80);"></font>

```javascript
for (let i = startIndex;i <= n;i++) {
  path.push(i);
  backtracking(n, k, i + 1);
  path.pop();
}
```



<font style="color:rgb(44, 62, 80);">这个遍历的范围是可以剪枝优化的，怎么优化呢？</font>

<font style="color:rgb(44, 62, 80);">来举一个例子，n = 4，k = 4的话，那么第一层for循环的时候，从元素2开始的遍历都没有意义了。 在第二层for循环，从元素3开始的遍历都没有意义了。</font>

<font style="color:rgb(44, 62, 80);">这么说有点抽象，如图所示：</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1659590350889-41fd30af-51d1-4869-b0a1-a572ea86ea0d.png)

<font style="color:rgb(44, 62, 80);">图中每一个节点（图中为矩形），就代表本层的一个for循环，那么每一层的for循环从第二个数开始遍历的话，都没有意义，都是无效遍历。</font>

**<font style="color:rgb(44, 62, 80);">所以，可以剪枝的地方就在递归中每一层的for循环所选择的起始位置</font>**<font style="color:rgb(44, 62, 80);">。</font>

**<font style="color:rgb(44, 62, 80);">如果for循环选择的起始位置之后的元素个数 已经不足 我们需要的元素个数了，那么就没有必要搜索了</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">注意代码中i，就是for循环里选择的起始位置。</font>

```javascript
for (let i = startIndex;i <= n;i++) {}

```



<font style="color:rgb(44, 62, 80);">接下来看一下优化过程如下：</font>

1. <font style="color:rgb(44, 62, 80);">已经选择的元素个数：path.size();</font>
2. <font style="color:rgb(44, 62, 80);">还需要的元素个数为: k - path.size();</font>
3. <font style="color:rgb(44, 62, 80);">在集合n中至多要从该起始位置 : n - (k - path.size()) + 1，开始遍历</font>
4. <font style="color:rgb(44, 62, 80);">为什么有个+1呢，因为包括起始位置，我们要是一个左闭的集合。</font>
5. <font style="color:rgb(44, 62, 80);">举个例子，n = 4，k = 3， 目前已经选取的元素为0（path.size为0），n - (k - 0) + 1 即 4 - ( 3 - 0) + 1 = 2。</font>

<font style="color:rgb(44, 62, 80);">从2开始搜索都是合理的，可以是组合[2, 3, 4]。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">所以优化之后的for循环是：</font>

```javascript

for (let i = startIndex;i <= n - (k - path.length) + 1;i++) {// i为本次搜索的起始位置

}
```

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">优化后整体代码如下：</font>

```javascript
 function combine(n, k) {
   const result = [];
   const path = [];

   const backtracking = function (n, k, startIndex) {
        if (path.length === k) {
            result.push([...path]);
            return;
        }

        for (let i = startIndex;i <= n - (k - path.length) + 1;i++) {
            path.push(i); // 处理节点
            backtracking(n, k, i + 1); // 递归
            path.pop(); // 回溯, 撤销处理的节点
        }
   }

   backtracking(n, k, 1);
   return result;
 }
```
