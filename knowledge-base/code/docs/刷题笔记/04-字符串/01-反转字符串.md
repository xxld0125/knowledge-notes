<font style="color:rgb(44, 62, 80);">编写一个函数，其作用是将输入的字符串反转过来。输入字符串以字符数组 char[] 的形式给出。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">不要给另外的数组分配额外的空间，你必须原地修改输入数组、使用 $O(1)$ 的额外空间解决这一问题。</font>

<font style="color:rgb(44, 62, 80);">你可以假设数组中的所有字符都是 ASCII 码表中的可打印字符。'</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">示例 1：

</font><font style="color:rgb(44, 62, 80);">输出：["o","l","l","e","h"]</font>

<font style="color:rgb(44, 62, 80);">示例 2：  
</font><font style="color:rgb(44, 62, 80);">输入：["H","a","n","n","a","h"]

#### 思路
<font style="color:rgb(44, 62, 80);">对于这道题目一些同学直接</font>`<font style="color:rgb(44, 62, 80);">reverse</font>`<font style="color:rgb(44, 62, 80);">，调一下直接完事了， 相信每一门编程语言都有这样的库函数。</font>

<font style="color:rgb(44, 62, 80);">如果这么做题的话，这样大家不会清楚反转字符串的实现原理了。</font>

<font style="color:rgb(44, 62, 80);">但是也不是说库函数就不能用，是要分场景的。</font>

<font style="color:rgb(44, 62, 80);">如果在现场面试中，我们什么时候使用库函数，什么时候不要用库函数呢？</font>

**<font style="color:rgb(44, 62, 80);">如果题目关键的部分直接用库函数就可以解决，建议不要使用库函数。</font>**

<font style="color:rgb(44, 62, 80);">毕竟面试官一定不是考察你对库函数的熟悉程度, </font>**<font style="color:rgb(44, 62, 80);">如果库函数仅仅是 解题过程中的一小部分，并且你已经很清楚这个库函数的内部实现原理的话，可以考虑使用库函数。</font>**

<font style="color:rgb(44, 62, 80);">建议大家平时在leetcode上练习算法的时候本着这样的原则去练习，这样才有助于我们对算法的理解。</font>

<font style="color:rgb(44, 62, 80);">不要沉迷于使用库函数一行代码解决题目之类的技巧，不是说这些技巧不好，而是说这些技巧可以用来娱乐一下。</font>

**<font style="color:#E8323C;">真正自己写的时候，要保证理解可以实现是相应的功能。</font>**





<font style="color:rgb(44, 62, 80);">在</font>[03-反转链表](https://www.yuque.com/u25370234/kb/bxx3vz)<font style="color:rgb(44, 62, 80);">中，使用了双指针的方法。那么反转字符串依然是使用双指针的方法，只不过对于字符串的反转，其实要比链表简单一些。</font>

<font style="color:rgb(44, 62, 80);">因为字符串也是一种数组，所以元素在内存中是连续分布，这就决定了反转链表和反转字符串方式上还是有所差异的。</font>

<font style="color:rgb(44, 62, 80);">对于字符串，我们定义两个指针（也可以说是索引下标），一个从字符串前面，一个从字符串后面，两个指针同时向中间移动，并交换元素。</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1652596777328-df185aaf-e968-4f62-af80-5e1d94fb4470.png)

  


**<font style="color:#E8323C;"></font>**

```javascript
/**
 * @param {character[]} s
 * @return {void} Do not return anything, modify s in-place instead.
 */
var reverseString = function(s) {
    const len = s.length;
    if (len < 2) return s;

    let l = 0,
        r = len - 1;
    
    while (l < r) {
        const tmp = s[l];

        s[l] = s[r];
        s[r] = tmp;

        l++;
        r--;
    }
    return s;
    
};
```
