### 一、[236. 二叉树的最近公共祖先](https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/)
<font style="color:rgb(44, 62, 80);">给定一个二叉树, 找到该树中两个指定节点的最近公共祖先。</font>

<font style="color:rgb(44, 62, 80);">百度百科中最近公共祖先的定义为：“对于有根树 T 的两个结点 p、q，最近公共祖先表示为一个结点 x，满足 x 是 p、q 的祖先且 x 的深度尽可能大（一个节点也可以是它自己的祖先）。”</font>

<font style="color:rgb(44, 62, 80);">例如，给定如下二叉树:  root = [3,5,1,6,2,0,8,null,null,7,4]</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1655354506193-78d7239d-014b-4d04-a46d-e0fb32d98ca3.png)

<font style="color:rgb(44, 62, 80);">示例 1: 输入: root = [3,5,1,6,2,0,8,null,null,7,4], p = 5, q = 1 输出: 3 解释: 节点 5 和节点 1 的最近公共祖先是节点 3。</font>

<font style="color:rgb(44, 62, 80);">示例 2: 输入: root = [3,5,1,6,2,0,8,null,null,7,4], p = 5, q = 4 输出: 5 解释: 节点 5 和节点 4 的最近公共祖先是节点 5。因为根据定义最近公共祖先节点可以为节点本身。</font>

<font style="color:rgb(44, 62, 80);">说明:</font>

+ <font style="color:rgb(44, 62, 80);">所有节点的值都是唯一的。</font>
+ <font style="color:rgb(44, 62, 80);">p、q 为不同节点且均存在于给定的二叉树中。</font>

<font style="color:rgb(44, 62, 80);"></font>

#### <font style="color:rgb(44, 62, 80);">思路</font>
<font style="color:rgb(44, 62, 80);">遇到这个题目首先想的是要是能自底向上查找就好了，这样就可以找到公共祖先了。那么二叉树如何可以自底向上查找呢？回溯啊，二叉树回溯的过程就是从低到上。后序遍历就是天然的回溯过程，最先处理的一定是叶子节点。</font>

<font style="color:rgb(44, 62, 80);">接下来就看如何判断一个节点是节点q和节点p的公共公共祖先呢。</font>

**<font style="color:rgb(44, 62, 80);">首先最容易想到的一个情况：如果找到一个节点，发现左子树出现结点p，右子树出现节点q，或者 左子树出现结点q，右子树出现节点p，那么该节点就是节点p和q的最近公共祖先。</font>**

**<font style="color:rgb(44, 62, 80);">但是很多人容易忽略一个情况，就是节点本身p(q)，它拥有一个子孙节点q(p)。</font>**

<font style="color:rgb(44, 62, 80);">使用后序遍历，回溯的过程，就是从低向上遍历节点，一旦发现满足第一种情况的节点，就是最近公共节点了。</font>

**<font style="color:rgb(44, 62, 80);">但是如果p或者q本身就是最近公共祖先呢？其实只需要找到一个节点是p或者q的时候，直接返回当前节点，无需继续递归子树。如果接下来的遍历中找到了后继节点满足第一种情况则修改返回值为后继节点，否则，继续返回已找到的节点即可。为什么满足第一种情况的节点一定是p或q的后继节点呢?大家可以仔细思考一下。</font>**

**<font style="color:rgb(44, 62, 80);"></font>**

+ 确定递归函数返回值及参数

<font style="color:rgb(44, 62, 80);">需要递归函数返回值，来告诉我们是否找到节点q或者p，那么返回值为bool类型就可以了。但我们还要返回最近公共节点，可以利用上题目中返回值是TreeNode，那么如果遇到p或者q，就把q或者p返回，返回值不为空，就说明找到了q或者p。</font>

```javascript
lowestCommonAncestor(root, p, q) {}
```

+ 确定终止条件

如果找到了节点p或q， 或者遇到空节点就返回。

```javascript
if (root === q || root === p || root === null) return root;
```

+ 确定单层递归逻辑

<font style="color:rgb(44, 62, 80);">值得注意的是 本题函数有返回值，是因为回溯的过程需要递归函数的返回值做判断，但本题我们依然要遍历树的所有节点。</font>

<font style="color:rgb(44, 62, 80);"></font><font style="color:rgb(44, 62, 80);">递归函数有返回值就是要遍历某一条边，但有返回值也要看如何处理返回值！</font>

<font style="color:rgb(44, 62, 80);">如果递归函数有返回值，如何区分要搜索一条边，还是搜索整个树呢？</font>

<font style="color:rgb(44, 62, 80);">搜索一条边的写法：</font>

```javascript
if (递归函数(root.left) return;

if (递归函数(root.right) return;
```

搜索整个树写法：

```javascript
left = 递归函数(root.left);
right = 递归函数(root.right);
left & right 的逻辑处理;
```

<font style="color:rgb(44, 62, 80);">看出区别了没？</font>

**<font style="color:rgb(44, 62, 80);">在递归函数有返回值的情况下：如果要搜索一条边，递归函数返回值不为空的时候，立刻返回，如果搜索整个树，直接用一个变量left、right接住返回值，这个left、right后序还有逻辑处理的需要，也就是后序遍历中处理中间节点的逻辑（也是回溯）</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">那么为什么要遍历整棵树呢？直观上来看，找到最近公共祖先，直接一路返回就可以了。</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1655355326630-82d9b472-0ac1-491d-9050-c61e5332c715.png)

<font style="color:rgb(44, 62, 80);">就像图中一样直接返回7，多美滋滋。</font>

<font style="color:rgb(44, 62, 80);">但事实上还要遍历根节点右子树（即使此时已经找到了目标节点了），也就是图中的节点4、15、20。</font>

<font style="color:rgb(44, 62, 80);">因为在如下代码的后序遍历中，如果想利用left和right做逻辑处理， 不能立刻返回，而是要等left与right逻辑处理完之后才能返回。</font>

```javascript
left = 递归函数(root.left);
right = 递归函数(root.right);
left & right 的逻辑处理;
```

所以<font style="color:rgb(44, 62, 80);">此时大家要知道我们要遍历整棵树。知道这一点，对本题就有一定深度的理解了。</font>

<font style="color:rgb(44, 62, 80);">那么先用left和right接住左子树和右子树的返回值，代码如下：</font>

```javascript
let left = lowestCommonAncestor(root.left, p, q);
let right = lowestCommonAncestor(root.right, p, q);
```

**<font style="color:rgb(44, 62, 80);">如果left 和 right都不为空，说明此时root就是最近公共节点。这个比较好理解</font>**

**<font style="color:rgb(44, 62, 80);">如果left为空，right不为空，就返回right，说明目标节点是通过right返回的，反之依然</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">这里有的同学就理解不了了，为什么left为空，right不为空，目标节点通过right返回呢？</font>

<font style="color:rgb(44, 62, 80);">如图：</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1655355608882-0efbfba9-b10a-4866-b737-ac396656538b.png)

<font style="color:rgb(44, 62, 80);">图中节点10的左子树返回null，右子树返回目标值7，那么此时节点10的处理逻辑就是把右子树的返回值（最近公共祖先7）返回上去！</font>

<font style="color:rgb(44, 62, 80);">这里点也很重要，可能刷过这道题目的同学，都不清楚结果究竟是如何从底层一层一层传到头结点的。</font>

<font style="color:rgb(44, 62, 80);">那么如果left和right都为空，则返回left或者right都是可以的，也就是返回空。</font>

<font style="color:rgb(44, 62, 80);">代码如下：</font>

```javascript
if (left === null && right !== null) return right;
else if (left !== null && right === null) return left;
else {
  return null;
}
```

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">那么寻找最小公共祖先，完整流程图如下：</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1655355808861-7ca01916-c3c6-4c2d-b757-5b9e80db23c5.png)

**<font style="color:rgb(44, 62, 80);">从图中，大家可以看到，我们是如何回溯遍历整棵二叉树，将结果返回给头结点的！</font>**

<font style="color:rgb(44, 62, 80);">整体代码如下：</font>

```javascript
var lowestCommonAncestor = function(root, p, q) {
    // 使用递归的方法
    // 需要从下到上，所以使用后序遍历
    // 1. 确定递归的函数
    const travelTree = function(root, p, q) {
        // 2. 确定递归终止条件
        if(root === null || root === p || root === q) {
            return root;
        }
        // 3. 确定递归单层逻辑
        let left = travelTree(root.left, p, q);
        let right = travelTree(root.right, p, q);
        if(left !== null&&right !== null) {
            return root;
        }
        if(left === null) {
            return right;
        }
        return left;
    }
   return  travelTree(root, p, q);
};
```



### 总结
<font style="color:rgb(44, 62, 80);">这道题目刷过的同学未必真正了解这里面回溯的过程，以及结果是如何一层一层传上去的。</font>

**<font style="color:rgb(44, 62, 80);">那么我给大家归纳如下三点</font>**<font style="color:rgb(44, 62, 80);">：</font>

1. <font style="color:rgb(44, 62, 80);">求最小公共祖先，需要从底向上遍历，那么二叉树，只能通过后序遍历（即：回溯）实现从低向上的遍历方式。</font>
2. <font style="color:rgb(44, 62, 80);">在回溯的过程中，必然要遍历整棵二叉树，即使已经找到结果了，依然要把其他节点遍历完，因为要使用递归函数的返回值（也就是代码中的</font>`<font style="color:rgb(44, 62, 80);">left</font>`<font style="color:rgb(44, 62, 80);">和</font>`<font style="color:rgb(44, 62, 80);">right</font>`<font style="color:rgb(44, 62, 80);">）做逻辑判断。</font>
3. <font style="color:rgb(44, 62, 80);">要理解如果返回值</font>`<font style="color:rgb(44, 62, 80);">left</font>`<font style="color:rgb(44, 62, 80);">为空，</font>`<font style="color:rgb(44, 62, 80);">right</font>`<font style="color:rgb(44, 62, 80);">不为空为什么要返回</font>`<font style="color:rgb(44, 62, 80);">right</font>`<font style="color:rgb(44, 62, 80);">，为什么可以用返回</font>`<font style="color:rgb(44, 62, 80);">right</font>`<font style="color:rgb(44, 62, 80);">传给上一层结果。</font>

<font style="color:rgb(44, 62, 80);">可以说这里每一步，都是有难度的，都需要对二叉树，递归和回溯有一定的理解。</font>

<font style="color:rgb(44, 62, 80);">本题没有给出迭代法，因为迭代法不适合模拟回溯的过程。理解递归的解法就够了。</font>
