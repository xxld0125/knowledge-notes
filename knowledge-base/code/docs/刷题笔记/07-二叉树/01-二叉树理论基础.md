### 二、二叉树的种类
<font style="color:rgb(44, 62, 80);"></font>

#### 1、满二叉树
<font style="color:rgb(44, 62, 80);">满二叉树：如果一棵二叉树只有度为0的结点和度为2的结点，并且度为0的结点在同一层上，则这棵二叉树为满二叉树。</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1652763224638-7039374e-5481-4486-9aa3-1032199f4bd9.png)

<font style="color:rgb(44, 62, 80);">这棵二叉树为满二叉树，也可以说深度为k，有2^k-1个节点的二叉树。</font>



#### 2、完全二叉树
<font style="color:rgb(44, 62, 80);">完全二叉树的定义如下：在完全二叉树中，除了最底层节点可能没填满外，其余每层节点数都达到最大值，并且最下面一层的节点都集中在该层最左边的若干位置。若最底层为第 h 层，则该层包含 1~ 2^(h-1)  个节点。</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1652763368926-f56b0332-6af5-446a-a485-71718d9f8669.png)

<font style="color:rgb(44, 62, 80);">相信不少同学最后一个二叉树是不是完全二叉树都中招了。</font>

**<font style="color:rgb(44, 62, 80);">之前我们刚刚讲过优先级队列其实是一个堆，堆就是一棵完全二叉树，同时保证父子节点的顺序关系。</font>**  


#### 3、二叉搜索树
<font style="color:rgb(44, 62, 80);">前面介绍的树，都没有数值的，而二叉搜索树是有数值的了，</font>**<font style="color:rgb(44, 62, 80);">二叉搜索树是一个有序树</font>**<font style="color:rgb(44, 62, 80);">。</font>

+ <font style="color:rgb(44, 62, 80);">若它的左子树不空，则左子树上所有结点的值均小于它的根结点的值；</font>
+ <font style="color:rgb(44, 62, 80);">若它的右子树不空，则右子树上所有结点的值均大于它的根结点的值；</font>
+ <font style="color:rgb(44, 62, 80);">它的左、右子树也分别为二叉排序树</font>

<font style="color:rgb(44, 62, 80);"></font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1652763486870-4ab77d76-81e7-4361-9221-a122cc4413de.png)



#### 4、平衡二叉搜索树
<font style="color:rgb(44, 62, 80);">平衡二叉搜索树：又被称为AVL（Adelson-Velsky and Landis）树，且具有以下性质：它是一棵空树或它的左右两个子树的高度差的绝对值不超过1，并且左右两个子树都是一棵平衡二叉树。</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1652763536823-3ed33661-792c-43bf-8b8e-8b9e620112fc.png)

<font style="color:rgb(44, 62, 80);">最后一棵 不是平衡二叉树，因为它的左右两个子树的高度差的绝对值超过了1。</font>

<font style="color:rgb(44, 62, 80);"></font>

### <font style="color:rgb(44, 62, 80);">三、二叉树的存储方式</font>
**<font style="color:rgb(44, 62, 80);">二叉树可以链式存储，也可以顺序存储。</font>**

<font style="color:rgb(44, 62, 80);">那么链式存储方式就用指针， 顺序存储的方式就是用数组。</font>

<font style="color:rgb(44, 62, 80);">顾名思义就是顺序存储的元素在内存是连续分布的，而链式存储则是通过指针把分布在散落在各个地址的节点串联一起。</font>



<font style="color:rgb(44, 62, 80);">链式存储如图：</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1652763716794-4f0a75b5-98ee-4d92-80c3-da930b007304.png)

  
	顺序存储其实就是用数组来存储二叉树, 存储方式如图:

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1652763783689-714ec54a-db1b-44c6-99bc-e53f3053c114.png)

<font style="color:rgb(44, 62, 80);">用数组来存储二叉树如何遍历的呢？</font>

**<font style="color:rgb(44, 62, 80);">如果父节点的数组下标是 i，那么它的左孩子就是 i * 2 + 1，右孩子就是 i * 2 + 2。</font>**

<font style="color:rgb(44, 62, 80);">但是用链式表示的二叉树，更有利于我们理解，所以一般我们都是用链式存储二叉树。</font>

**<font style="color:rgb(44, 62, 80);">所以大家要了解，用数组依然可以表示二叉树。</font>**

  
	

### 四、二叉树的遍历方式
<font style="color:rgb(44, 62, 80);">二叉树主要有两种遍历方式：</font>

1. <font style="color:rgb(44, 62, 80);">深度优先遍历：先往深走，遇到叶子节点再往回走。</font>
2. <font style="color:rgb(44, 62, 80);">广度优先遍历：一层一层的去遍历。</font>

<font style="color:rgb(44, 62, 80);"></font>

**<font style="color:rgb(44, 62, 80);">这两种遍历是图论中最基本的两种遍历方式</font>**<font style="color:rgb(44, 62, 80);">，后面在介绍图论的时候 还会介绍到。</font>

<font style="color:rgb(44, 62, 80);">那么从深度优先遍历和广度优先遍历进一步拓展，才有如下遍历方式：</font>

<font style="color:rgb(44, 62, 80);"></font>

+ <font style="color:rgb(44, 62, 80);">深度优先遍历</font>
    - <font style="color:rgb(44, 62, 80);">前序遍历（递归法，迭代法）</font>
    - <font style="color:rgb(44, 62, 80);">中序遍历（递归法，迭代法）</font>
    - <font style="color:rgb(44, 62, 80);">后序遍历（递归法，迭代法）</font>
+ <font style="color:rgb(44, 62, 80);">广度优先遍历</font>
    - <font style="color:rgb(44, 62, 80);">层次遍历（迭代法）</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">在深度优先遍历中：有三个顺序，前中后序遍历， 有同学总分不清这三个顺序，经常搞混，我这里教大家一个技巧。</font>

**<font style="color:rgb(44, 62, 80);">这里前中后，其实指的就是中间节点的遍历顺序</font>**<font style="color:rgb(44, 62, 80);">，只要大家记住 前中后序指的就是中间节点的位置就可以了。</font>

<font style="color:rgb(44, 62, 80);">看如下中间节点的顺序，就可以发现，中间节点的顺序就是所谓的遍历方式</font>

+ <font style="color:rgb(44, 62, 80);">前序遍历：中左右</font>
+ <font style="color:rgb(44, 62, 80);">中序遍历：左中右</font>
+ <font style="color:rgb(44, 62, 80);">后序遍历：左右中</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">大家可以对着如下图，看看自己理解的前后中序有没有问题。</font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1652763990517-8918fc73-fc04-48c8-a568-ec7dd0de296c.png)

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">最后再说一说二叉树中深度优先和广度优先遍历实现方式，我们做二叉树相关题目，经常会使用递归的方式来实现深度优先遍历，也就是实现前中后序遍历，使用递归是比较方便的。</font>

**<font style="color:rgb(44, 62, 80);">之前我们讲栈与队列的时候，就说过栈其实就是递归的一种是实现结构</font>**<font style="color:rgb(44, 62, 80);">，也就说前中后序遍历的逻辑其实都是可以借助栈使用非递归的方式来实现的。</font>

<font style="color:rgb(44, 62, 80);">而广度优先遍历的实现一般使用队列来实现，这也是队列先进先出的特点所决定的，因为需要先进先出的结构，才能一层一层的来遍历二叉树。</font>

**<font style="color:rgb(44, 62, 80);">这里其实我们又了解了栈与队列的一个应用场景了。</font>**



### 五、二叉树的定义
<font style="color:rgb(44, 62, 80);">刚刚我们说过了二叉树有两种存储方式顺序存储，和链式存储，顺序存储就是用数组来存，这个定义没啥可说的，我们来看看链式存储的二叉树节点的定义方式。</font>

```bash
function TreeNode(val, left, right) {
    this.val = (val === undefined ? 0 : val)
    this.left = (left === undefined ? null : left)
    this.right = (right === undefined ? null : right)
}
```

<font style="color:rgb(44, 62, 80);">大家会发现二叉树的定义 和链表是差不多的，相对于链表 ，二叉树的节点里多了一个指针， 有两个指针，指向左右孩子。</font>

<font style="color:rgb(44, 62, 80);">这里要提醒大家要注意二叉树节点定义的书写方式。</font>

**<font style="color:rgb(44, 62, 80);">在现场面试的时候 面试官可能要求手写代码，所以数据结构的定义以及简单逻辑的代码一定要锻炼白纸写出来。</font>**

<font style="color:rgb(44, 62, 80);">因为我们在刷leetcode的时候，节点的定义默认都定义好了，真到面试的时候，需要自己写节点定义的时候，有时候会一脸懵逼</font>
