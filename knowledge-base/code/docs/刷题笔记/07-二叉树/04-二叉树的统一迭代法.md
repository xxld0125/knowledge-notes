### 一、统一迭代法
在[02-二叉树的递归遍历](https://www.yuque.com/u25370234/kb/ih5mpo)中,用递归的方式,实现了二叉树前中后序的遍历<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);">在</font>[03-二叉树的迭代遍历](https://www.yuque.com/u25370234/kb/smsmle)<font style="color:rgb(44, 62, 80);">中用栈实现了二叉树前后中序的迭代遍历。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">之后我们发现</font>**<font style="color:rgb(44, 62, 80);">迭代法实现的先中后序，其实风格也不是那么统一，除了先序和后序，有关联，中序完全就是另一个风格了，一会用栈遍历，一会又用指针来遍历。</font>**

<font style="color:rgb(44, 62, 80);">实践过的同学，也会发现使用迭代法实现先中后序遍历，很难写出统一的代码，不像是递归法，实现了其中的一种遍历方式，其他两种只要稍稍改一下节点顺序就可以了。</font>

<font style="color:rgb(44, 62, 80);">其实</font>**<font style="color:rgb(44, 62, 80);">针对三种遍历方式，使用迭代法是可以写出统一风格的代码！</font>**

**<font style="color:rgb(44, 62, 80);"></font>**

**<font style="color:rgb(44, 62, 80);">重头戏来了，接下来介绍一下统一写法。</font>**

**<font style="color:rgb(44, 62, 80);"></font>**

<font style="color:rgb(44, 62, 80);">我们以中序遍历为例，在</font>[03-二叉树的迭代遍历](https://www.yuque.com/u25370234/kb/smsmle)<font style="color:rgb(44, 62, 80);">中提到说使用栈的话，</font>**<font style="color:rgb(44, 62, 80);">无法同时解决访问节点（遍历节点）和处理节点（将元素放进结果集）不一致的情况</font>**<font style="color:rgb(44, 62, 80);">。 </font>

**<font style="color:rgb(44, 62, 80);">那我们就将访问的节点放入栈中，把要处理的节点也放入栈中但是要做标记。</font>**

<font style="color:rgb(44, 62, 80);">如何标记呢，</font>**<font style="color:rgb(44, 62, 80);">就是要处理的节点放入栈之后，紧接着放入一个空指针作为标记。</font>**<font style="color:rgb(44, 62, 80);"> 这种方法也可以叫做</font>**<font style="color:rgb(44, 62, 80);">标记法</font>**<font style="color:rgb(44, 62, 80);">。</font>

<font style="color:rgb(44, 62, 80);"></font>

### 二、迭代法中序遍历
```javascript
//  中序遍历：左中右
//  压栈顺序：右中左
 
var inorderTraversal = function(root, res = []) {
    const stack = [];
    if (root) stack.push(root);
    while(stack.length) {
        const node = stack.pop(); // 将该节点弹出，避免重复操作，下面再将右中左节点添加到栈中
        if(!node) {
            res.push(stack.pop().val); // 重新取出栈中元素并加入到结果集
            continue; 
        }
        if (node.right) stack.push(node.right); // 添加右节点（空节点不入栈）
        stack.push(node); // 添加中节点
        stack.push(null); // 中节点访问过，但是还没有处理，加入空节点做为标记。
        if (node.left) stack.push(node.left); // 添加左节点（空节点不入栈）
    };
    return res;
};
```

<font style="color:rgb(44, 62, 80);">可以看出我们将访问的节点直接加入到栈中，但如果是处理的节点则后面放入一个空节点， 这样只有空节点弹出的时候，才将下一个节点放进结果集。</font>

<font style="color:rgb(44, 62, 80);">此时我们再来看前序遍历代码。</font>

<font style="color:rgb(44, 62, 80);"></font>

### 三、迭代法前序遍历
```javascript
//  中序遍历：左中右
//  压栈顺序：右中左
 
var inorderTraversal = function(root, res = []) {
    const res = [];
    const stack = [];
    if (root) stack.push(root);
    while (stack.length) {
        const node = stack.pop();
        if (!node) {
            res.push(stack.pop().val);
            continue;
        }
        if (node.right) stack.push(node.right);
        if (node.left) stack.push(node.left);
        stack.push(node);
        stack.push(null);
    }
    return res;
};
```



### 四、迭代法后序遍历
```javascript
//  中序遍历：左中右
//  压栈顺序：右中左
 
var inorderTraversal = function(root, res = []) {
    const res = [];
    const stack = [];
    if (root) stack.push(root);
    while (stack.length) {
        const node = stack.pop();
        if (!node) {
            res.push(stack.pop().val);
            continue;
        }
        stack.push(node);
        stack.push(null);

        if (node.right) stack.push(node.right);
        if (node.left) stack.push(node.left);
    }
    return res;
};
```
