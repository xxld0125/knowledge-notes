### [03-二叉树的迭代遍历](https://www.yuque.com/u25370234/kb/smsmle)
递归也可以使用栈实现,  也就是所谓的迭代法<font style="color:rgb(44, 62, 80);">。介绍了前中后序递归如何用迭代法实现。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">递归和迭代谁优谁劣?</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">从时间复杂度上其实迭代法和递归法差不多（在不考虑函数调用开销和函数调用产生的堆栈开销），但是空间复杂度上，递归开销会大一些，因为递归需要系统堆栈存参数返回值等等。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">递归更容易让程序员理解，但收敛不好，容易栈溢出。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">这么说吧，递归是方便了程序员，难为了机器（各种保存参数，各种进栈出栈）。</font>

**<font style="color:rgb(44, 62, 80);">在实际项目开发的过程中我们是要尽量避免递归！因为项目代码参数、调用关系都比较复杂，不容易控制递归深度，甚至会栈溢出。</font>**



### [04-二叉树的统一迭代法](https://www.yuque.com/u25370234/kb/xp51r2)

<font style="color:rgb(44, 62, 80);">此时又多了一种前中后序的迭代写法，那么有同学问了：前中后序迭代法是不是一定要统一来写，这样才算是规范。</font>

<font style="color:rgb(44, 62, 80);">其实没必要，还是自己感觉哪一种更好记就用哪种。</font>

<font style="color:rgb(44, 62, 80);">但是</font>**<font style="color:rgb(44, 62, 80);">一定要掌握前中后序一种迭代的写法，并不因为某种场景的题目一定要用迭代，而是现场面试的时候，面试官看你顺畅的写出了递归，一般会进一步考察能不能写出相应的迭代。</font>**
