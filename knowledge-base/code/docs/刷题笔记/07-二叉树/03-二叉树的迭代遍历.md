### 一、迭代遍历
<font style="color:rgb(44, 62, 80);">为什么可以用迭代法（非递归的方式）来实现二叉树的前后中序遍历呢？</font>

**<font style="color:rgb(44, 62, 80);">递归的实现就是：每一次递归调用都会把函数的局部变量、参数值和返回地址等压入调用栈中</font>**<font style="color:rgb(44, 62, 80);">，然后递归返回的时候，从栈顶弹出上一次递归的各项参数，所以这就是递归为什么可以返回上一层位置的原因。</font>

<font style="color:rgb(44, 62, 80);">此时大家应该知道我们用栈也可以是实现二叉树的前后中序遍历了。</font>

<font style="color:rgb(44, 62, 80);"></font>

### 二、前序遍历(迭代法)
<font style="color:rgb(44, 62, 80);">前序遍历是中左右，每次先处理的是中间节点，那么先将根节点放入栈中，然后将右孩子加入栈，再加入左孩子。</font>

<font style="color:rgb(44, 62, 80);">为什么要先加入 右孩子，再加入左孩子呢？ 因为这样出栈的时候才是中左右的顺序。</font>

![](https://cdn.nlark.com/yuque/0/2022/gif/25743026/1652792730369-3809f7cc-7bc0-4270-bd7a-7e8d97047135.gif)

```javascript
// 入栈 右 -> 左
// 出栈 中 -> 左 -> 右
var preorderTraversal = function(root, res = []) {
    if(!root) return res;
    const stack = [root];
    let cur = null;
    while(stack.length) {
        cur = stack.pop(); // 中
        res.push(cur.val);
        cur.right && stack.push(cur.right); // 右（空节点不入栈）
        cur.left && stack.push(cur.left); // 左（空节点不入栈）
    }
    return res;
};
```

<font style="color:rgb(44, 62, 80);">此时会发现貌似使用迭代法写出前序遍历并不难，确实不难。</font>

**<font style="color:rgb(44, 62, 80);">此时是不是想改一点前序遍历代码顺序就把中序遍历搞出来了？</font>**

**<font style="color:rgb(44, 62, 80);"></font>**

<font style="color:rgb(44, 62, 80);">其实还真不行！</font>**<font style="color:rgb(44, 62, 80);">用迭代法写中序遍历的时候，会发现套路又不一样了，目前的前序遍历的逻辑无法直接应用到中序遍历上。</font>**



### 三、中序遍历(迭代法)
<font style="color:rgb(44, 62, 80);">为了解释清楚，我说明一下 刚刚在迭代的过程中，其实我们有两个操作：</font>

    1. **<font style="color:rgb(44, 62, 80);">处理：将元素放进result数组中</font>**
    2. **<font style="color:rgb(44, 62, 80);">访问：遍历节点</font>**

**<font style="color:rgb(44, 62, 80);"></font>**

<font style="color:rgb(44, 62, 80);">分析一下为什么刚刚写的前序遍历的代码，不能和中序遍历通用呢，因为前序遍历的顺序是中左右，先访问的元素是中间节点，要处理的元素也是中间节点，所以刚刚才能写出相对简洁的代码，</font>**<font style="color:rgb(44, 62, 80);">因为要访问的元素和要处理的元素顺序是一致的，都是中间节点。</font>**

<font style="color:rgb(44, 62, 80);">那么再看看中序遍历，中序遍历是左中右，先访问的是二叉树顶部的节点，然后一层一层向下访问，直到到达树左面的最底部，再开始处理节点（也就是在把节点的数值放进result数组中），这就造成了</font>**<font style="color:rgb(44, 62, 80);">处理顺序和访问顺序是不一致的。</font>**

<font style="color:rgb(44, 62, 80);">那么</font>**<font style="color:rgb(44, 62, 80);">在使用迭代法写中序遍历，就需要借用指针的遍历来帮助访问节点，栈则用来处理节点上的元素。</font>**

```javascript
// 入栈 左 -> 右
// 出栈 左 -> 中 -> 右
s
var inorderTraversal = function(root, res = []) {
    const stack = [];
    let cur = root;
    while(stack.length || cur) {
        if(cur) { // 指针来访问节点，访问到最底层
            stack.push(cur); // 将访问的节点放进栈
            // 左
            cur = cur.left; // 左
        } else {
            // --> 弹出 中
            cur = stack.pop(); // 从栈里弹出的数据，就是要处理的数据（放进result数组里的数据）
            res.push(cur.val); // 中
            // 右
            cur = cur.right; // 右
        }
    };
    return res;
};
```



### 四、后序遍历(迭代法)
<font style="color:rgb(44, 62, 80);">再来看后序遍历，先序遍历是中左右，后续遍历是左右中，那么我们只需要调整一下先序遍历的代码顺序，就变成中右左的遍历顺序，然后在反转result数组，输出的结果顺序就是左右中了</font>**<font style="color:rgb(44, 62, 80);">。</font>**

**<font style="color:rgb(44, 62, 80);"></font>**

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1652793859769-60596174-e26e-42f3-b068-56e50a1e71c0.png)

**<font style="color:rgb(44, 62, 80);">所以后序遍历只需要前序遍历的代码稍作修改就可以了，代码如下：</font>**

**<font style="color:rgb(44, 62, 80);"></font>**

```javascript
// 入栈 左 -> 右
// 出栈 中 -> 右 -> 左 结果翻转

var postorderTraversal = function(root, res = []) {
    if (!root) return [];
    const res = [];
    const stack = [root];
    let cur = null;
    while (stack.length) {
        cur = stack.pop();
        res.push(cur.val);
        // 相对于前序遍历，这更改一下入栈顺序 （空节点不入栈）
        cur.left && stack.push(cur.left);
        cur.right && stack.push(cur.right);
    }
    // 将结果反转之后就是左右中的顺序了
    return res.reverse();
};
```



### 五、总结
<font style="color:rgb(44, 62, 80);">此时我们用迭代法写出了二叉树的前后中序遍历，大家可以看出前序和中序是完全两种代码风格，并不像递归写法那样代码稍做调整，就可以实现前后中序。</font>

**<font style="color:rgb(44, 62, 80);">这是因为前序遍历中访问节点（遍历节点）和处理节点（将元素放进result数组中）可以同步处理，但是中序就无法做到同步！</font>**

<font style="color:rgb(44, 62, 80);">上面这句话，可能一些同学不太理解，建议自己亲手用迭代法，先写出来前序，再试试能不能写出中序，就能理解了。</font>
