### 一、[501. 二叉搜索树中的众数](https://leetcode.cn/problems/find-mode-in-binary-search-tree/)
<font style="color:rgb(44, 62, 80);">给定一个有相同值的二叉搜索树（BST），找出 BST 中的所有众数（出现频率最高的元素）。</font>

<font style="color:rgb(44, 62, 80);">假定 BST 有如下定义：</font>

+ <font style="color:rgb(44, 62, 80);">结点左子树中所含结点的值小于等于当前结点的值</font>
+ <font style="color:rgb(44, 62, 80);">结点右子树中所含结点的值大于等于当前结点的值</font>
+ <font style="color:rgb(44, 62, 80);">左子树和右子树都是二叉搜索树</font>

<font style="color:rgb(44, 62, 80);">例如：</font>

<font style="color:rgb(44, 62, 80);">给定 BST [1,null,2,2],</font>

<font style="color:rgb(44, 62, 80);">返回[2].</font>

<font style="color:rgb(44, 62, 80);">提示：如果众数超过1个，不需考虑输出顺序</font>

<font style="color:rgb(44, 62, 80);">进阶：你可以不使用额外的空间吗？（假设由递归产生的隐式调用栈的开销不被计算在内）</font>

#### 思路
<font style="color:rgb(44, 62, 80);">如果不是二叉搜索树，最直观的方法一定是把这个树都遍历了，用map统计频率，把频率排个序，最后取前面高频的元素的集合。</font>

<font style="color:rgb(44, 62, 80);"></font>

<font style="color:rgb(44, 62, 80);">是二叉搜索树的话, 那他的中序遍历就是有序的。</font>

<font style="color:rgb(44, 62, 80);"></font>

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1655217580933-8382de77-58fa-4ac6-876b-b3ed77226201.png)



<font style="color:rgb(44, 62, 80);">遍历有序数组的元素出现频率，从头遍历，那么一定是相邻两个元素作比较，然后就把出现频率最高的元素输出就可以了。</font>

<font style="color:rgb(44, 62, 80);">关键是在有序数组上的话，好搞，在树上怎么搞呢？</font>

<font style="color:rgb(44, 62, 80);">这就考察对树的操作了。</font>

<font style="color:rgb(44, 62, 80);">在</font>[25-二叉搜索树的最小绝对差](https://www.yuque.com/u25370234/kb/rc7vqk)<font style="color:rgb(44, 62, 80);">中我们就使用了</font>`<font style="color:rgb(44, 62, 80);">pre</font>`<font style="color:rgb(44, 62, 80);">指针和</font>`<font style="color:rgb(44, 62, 80);">cur</font>`<font style="color:rgb(44, 62, 80);">指针的技巧，这次又用上了。</font>

<font style="color:rgb(44, 62, 80);">弄一个指针指向前一个节点，这样每次</font>`<font style="color:rgb(44, 62, 80);">cur</font>`<font style="color:rgb(44, 62, 80);">（当前节点）才能和</font>`<font style="color:rgb(44, 62, 80);">pre</font>`<font style="color:rgb(44, 62, 80);">（前一个节点）作比较。</font>

<font style="color:rgb(44, 62, 80);">而且初始化的时候</font>`<font style="color:rgb(44, 62, 80);">pre = null</font>`<font style="color:rgb(44, 62, 80);">，这样当</font>`<font style="color:rgb(44, 62, 80);">pre</font>`<font style="color:rgb(44, 62, 80);">为</font>`<font style="color:rgb(44, 62, 80);">null</font>`<font style="color:rgb(44, 62, 80);">时候，我们就知道这是比较的第一个元素。</font>

```javascript
if (pre === null) { // 第一个节点
  count = 1; // 频率为1
} else if (pre.val === cur.val) { // 与前一个节点数值相同
  count++;
} else { // 与前一个节点数值不同
  count = 1
}
pre = cur; // 更新上一个节点
```

<font style="color:rgb(44, 62, 80);">此时又有问题了，因为要求最大频率的元素集合（注意是集合，不是一个元素，可以有多个众数），如果是数组上大家一般怎么办？</font>

<font style="color:rgb(44, 62, 80);">应该是先遍历一遍数组，找出最大频率（</font>`<font style="color:rgb(44, 62, 80);">maxCount</font>`<font style="color:rgb(44, 62, 80);">），然后再重新遍历一遍数组把出现频率为</font>`<font style="color:rgb(44, 62, 80);">maxCount</font>`<font style="color:rgb(44, 62, 80);">的元素放进集合。（因为众数有多个）</font>

<font style="color:rgb(44, 62, 80);">这种方式遍历了两遍数组。</font>

<font style="color:rgb(44, 62, 80);">那么我们遍历两遍二叉搜索树，把众数集合算出来也是可以的。</font>

<font style="color:rgb(44, 62, 80);">但这里其实只需要遍历一次就可以找到所有的众数。</font>

<font style="color:rgb(44, 62, 80);">那么如何只遍历一遍呢？</font>

<font style="color:rgb(44, 62, 80);">如果 频率</font>`<font style="color:rgb(44, 62, 80);">count</font>`<font style="color:rgb(44, 62, 80);"> 等于 </font>`<font style="color:rgb(44, 62, 80);">maxCount</font>`<font style="color:rgb(44, 62, 80);">（最大频率），当然要把这个元素加入到结果集中（以下代码为result数组），代码如下：</font>



```javascript
if (count === maxCount) { // 如果和最大值相同, 放入res中
  res.push(cur.val)
}
```

<font style="color:rgb(44, 62, 80);">是不是感觉这里有问题，result怎么能轻易就把元素放进去了呢，万一，这个maxCount此时还不是真正最大频率呢。</font>

<font style="color:rgb(44, 62, 80);">所以下面要做如下操作：</font>

<font style="color:rgb(44, 62, 80);">频率</font>`<font style="color:rgb(44, 62, 80);">count</font>`<font style="color:rgb(44, 62, 80);"> 大于 </font>`<font style="color:rgb(44, 62, 80);">maxCount</font>`<font style="color:rgb(44, 62, 80);">的时候，不仅要更新</font>`<font style="color:rgb(44, 62, 80);">maxCount</font>`<font style="color:rgb(44, 62, 80);">，而且要清空结果集（以下代码为re s数组），因为结果集之前的元素都失效了。</font>

```javascript
if (count > maxCount) { // 如果计数大于最大值
  maxCount = count; // 更新最大频率
  res = []; // 清空res
  res.push(cur.val);
}
```



#### 递归
##### 1.不是二叉搜索树
比较简单,略





##### 2.是二叉树搜索树
借助数组

```javascript
/**
 * Definition for a binary tree node.
 * function TreeNode(val, left, right) {
 *     this.val = (val===undefined ? 0 : val)
 *     this.left = (left===undefined ? null : left)
 *     this.right = (right===undefined ? null : right)
 * }
 */
/**
 * @param {TreeNode} root
 * @return {number[]}
 */
var findMode = function(root) {
    // 一、递归-借助数组
    const res = {};

    const dfs = node => {
        node.left && dfs(node.left);

        
        if (res[node.val]) {
            res[node.val]++;
        } else {
            res[node.val] = 1;
        }

        node.right && dfs(node.right);
    }

    dfs(root);

    let max = -Infinity;
    let arr = [];

    for (let i in res) {
        if (res.hasOwnProperty(i)) {
            if (res[i] === max) {
              arr.push(i);
            }
          
            if (res[i] > max) {
              max = res[i];
              arr = [i];
            }
        }
    }

    return arr;
};
```



```javascript
/**
 * Definition for a binary tree node.
 * function TreeNode(val, left, right) {
 *     this.val = (val===undefined ? 0 : val)
 *     this.left = (left===undefined ? null : left)
 *     this.right = (right===undefined ? null : right)
 * }
 */
/**
 * @param {TreeNode} root
 * @return {number[]}
 */
var findMode = function(root) {
    let pre = null,
        max = 0,
        count = 0,
        res = [];

    const dfs = node => {
        node.left && dfs(node.left);

        if (pre === null) {
            count = 1;
        } else if (pre.val === node.val){
            count++;
        } else {
            count = 1;
        }

        if (count === max) {
            res.push(node.val);
        } else if (count > max) {
            max = count;
            res = [node.val];
        }

        pre = node;

        node.right && dfs(node.right);
    }

    dfs(root);

    return res;
};
```



#### 迭代
```javascript
/**
 * Definition for a binary tree node.
 * function TreeNode(val, left, right) {
 *     this.val = (val===undefined ? 0 : val)
 *     this.left = (left===undefined ? null : left)
 *     this.right = (right===undefined ? null : right)
 * }
 */
/**
 * @param {TreeNode} root
 * @return {number[]}
 */
var findMode = function(root) {
    let res = [],
        pre = null,
        count = 0,
        max = 0;

    const stack = [root];

    while (stack.length) {
        let node = stack.pop();
        if (node === null) {
            node = stack.pop();

            if (pre === null) {
                count = 1;
            } else if (pre.val === node.val) {
                count++;
            } else {
                count = 1;
            }

            pre = node;

            if (count === max) {
                res.push(node.val);
            } else if (count > max) {
                max = count;
                res = [node.val];
            }

        } else {
            node.right && stack.push(node.right);

            stack.push(node);
            stack.push(null);

            node.left && stack.push(node.left)
        }
    }
    return res;
};
```





### 总结
<font style="color:rgb(44, 62, 80);">本题在递归法中，我给出了如果是普通二叉树，应该怎么求众数。</font>

<font style="color:rgb(44, 62, 80);">知道了普通二叉树的做法时候，我再进一步给出二叉搜索树又应该怎么求众数，这样鲜明的对比，相信会对二叉树又有更深层次的理解了。</font>

<font style="color:rgb(44, 62, 80);">在递归遍历二叉搜索树的过程中，我还介绍了一个统计最高出现频率元素集合的技巧， 要不然就要遍历两次二叉搜索树才能把这个最高出现频率元素的集合求出来。</font>

**<font style="color:rgb(44, 62, 80);">为什么没有这个技巧一定要遍历两次呢？ 因为要求的是集合，会有多个众数，如果规定只有一个众数，那么就遍历一次稳稳的了。</font>**
