<font style="background-color:rgba(255, 255, 255, 0);">梳理本地的前端项目如何部署到服务器上。</font>

#### <font style="background-color:rgba(255, 255, 255, 0);">编写Dockerfile文件</font>
<font style="background-color:rgba(255, 255, 255, 0);">Dockerfile文件中定义了镜像构建所需的基础镜像、环境变量、代码拷贝、依赖安装等相关内容，具体内容根据实际情况进行修改。</font>

#### <font style="background-color:rgba(255, 255, 255, 0);">构建Docker镜像</font>
<font style="background-color:rgba(255, 255, 255, 0);">使用docker build命令，根据Dockerfile文件构建出一个Docker镜像。例如，如果Dockerfile文件名为Dockerfile，则可以通过以下命令进行构建：</font>

```plain
arduinoCopy code
docker build -t my-image .
```

<font style="background-color:rgba(255, 255, 255, 0);">其中，</font>**<font style="background-color:rgba(255, 255, 255, 0);">-t</font>**<font style="background-color:rgba(255, 255, 255, 0);">参数指定了构建的镜像名称，</font>**<font style="background-color:rgba(255, 255, 255, 0);">.</font>**<font style="background-color:rgba(255, 255, 255, 0);">代表当前目录，表示以当前目录下的</font>

#### <font style="background-color:rgba(255, 255, 255, 0);">打包镜像</font>
<font style="background-color:rgba(255, 255, 255, 0);">在本地使用</font>**<font style="background-color:rgba(255, 255, 255, 0);">docker save</font>**<font style="background-color:rgba(255, 255, 255, 0);">命令将要上传的 Docker 镜像保存为 tar 文件，例如：</font>

```plain
docker save -o my_image.tar my_image:latest
```

#### <font style="background-color:rgba(255, 255, 255, 0);">将镜像压缩包上传到服务器</font>
<font style="background-color:rgba(255, 255, 255, 0);">使用 SCP 命令将镜像文件传输到服务器上，例如：</font>

```plain
scp my_image.tar user@server:/path/to/folder/
```

<font style="background-color:rgba(255, 255, 255, 0);">其中，</font>**<font style="background-color:rgba(255, 255, 255, 0);">user</font>**<font style="background-color:rgba(255, 255, 255, 0);">为服务器用户名，</font>**<font style="background-color:rgba(255, 255, 255, 0);">server</font>**<font style="background-color:rgba(255, 255, 255, 0);">为服务器地址，</font>**<font style="background-color:rgba(255, 255, 255, 0);">/path/to/folder/</font>**<font style="background-color:rgba(255, 255, 255, 0);">为要传输到的服务器文件夹路径。</font>

#### <font style="background-color:rgba(255, 255, 255, 0);">解压镜像压缩包</font>
<font style="background-color:rgba(255, 255, 255, 0);">在服务器上使用</font>**<font style="background-color:rgba(255, 255, 255, 0);">docker load</font>**<font style="background-color:rgba(255, 255, 255, 0);">命令加载镜像，例如：</font>

```plain
docker load -i /path/to/folder/my_image.tar
```

<font style="background-color:rgba(255, 255, 255, 0);">其中，</font>**<font style="background-color:rgba(255, 255, 255, 0);">/path/to/folder/</font>**<font style="background-color:rgba(255, 255, 255, 0);">为上传镜像文件的文件夹路径。</font>

#### <font style="background-color:rgba(255, 255, 255, 0);">运行Docker容器</font>
<font style="background-color:rgba(255, 255, 255, 0);">使用docker run命令，启动容器并运行镜像中的程序。例如，如果容器中的程序运行在3000端口，可以使用以下命令启动容器：</font>

```plain
docker run -p 3000:3000 my-image
```

<font style="background-color:rgba(255, 255, 255, 0);"></font>

<font style="background-color:rgba(255, 255, 255, 0);">这样就简单的将本地前端项目, 部署到服务器上了，可以通过服务器域名/ip: 3000访问前端项目，</font>

<font style="background-color:rgba(255, 255, 255, 0);">如果不想通过端口，或需要配置其他，需要再服务器上配置Nginx</font>
