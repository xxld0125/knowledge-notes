<font style="color:rgb(77, 77, 77);">管道有两个典型的应用场景:</font>

+ **转换**<font style="color:rgb(77, 77, 77);">：管道将输入数据转换为所需的数据输出(例如，将字符串转换为整数)</font>
+ **验证**<font style="color:rgb(77, 77, 77);">：对输入数据进行验证，如果验证成功继续传递; 验证失败则抛出异常</font>

<font style="color:rgb(77, 77, 77);"></font>

`Nest`<font style="color:rgb(77, 77, 77);"> 自带八个开箱即用的管道，即</font>

+ ValidationPipe
+ ParseIntPipe
+ ParseBoolPipe
+ ParseArrayPipe
+ ParseUUIDPipe
+ DefaultValuePipe
+ ParseEnumPipe
+ ParseFloatPipe

<font style="color:rgb(79, 79, 79);"></font>

#### <font style="color:rgb(79, 79, 79);">一、数据转换</font>
<font style="color:rgb(79, 79, 79);">我们接受的动态参数希望是一个</font>`<font style="color:rgb(79, 79, 79);">number</font>`<font style="color:rgb(79, 79, 79);"> 类型 现在是</font>`<font style="color:rgb(79, 79, 79);">string</font>`<font style="color:rgb(79, 79, 79);"> ，可以通过</font>`ParseIntPipe`做数据类型转换。

```typescript
import {
  Controller,
  Get,
  Param,
  ParseIntPipe,
  ParseUUIDPipe,
} from '@nestjs/common';
import { PipeService } from './pipe.service';

@Controller('pipe')
export class PipeController {
  constructor(private readonly pipeService: PipeService) {}

  @Get(':id')
  findOne(@Param('id', ParseIntPipe) id: number) {
    console.log('id', typeof id);
    return this.pipeService.findOne(+id);
  }
}

```



<font style="color:rgb(79, 79, 79);">我们接受的动态参数希望是一个uuid 类型 现在是</font>`<font style="color:rgb(79, 79, 79);">string</font>`<font style="color:rgb(79, 79, 79);"> ，可以通过</font>`ParseUUIDPipe`做数据类型转换。

`<font style="color:rgb(77, 77, 77);">npm install uuid -S</font>`

`<font style="color:rgb(77, 77, 77);">npm install @types/uuid -D</font>`

```typescript
import {

  }
}

```

#### 二、验证DTO
##### 1、自定义验证
安装验证包 `npm i --save class-validator class-transformer`



自定义数据验证逻辑。

```typescript
import { IsNotEmpty, IsString } from 'class-validator';

export class CreatePipeDto {
  @IsNotEmpty() // 验证是否为空
  @IsString() // 验证是否为字符串
  name: string;

  @IsNotEmpty()
  age: number;
}

```



在`controller`中使用管道和定义类型。

```typescript
import {
  Controller,
  Body,
  Post,
} from '@nestjs/common';
import { PipeService } from './pipe.service';
import { CreatePipeDto } from './dto/create-pipe.dto';
import { PipePipe } from './pipe.pipe';

@Controller('pipe')
export class PipeController {
  constructor(private readonly pipeService: PipeService) {}

  @Post()
  create(@Body(PipePipe) createPipeDto: CreatePipeDto) {
    return this.pipeService.create(createPipeDto);
  }
}

```



实现验证错误信息校验逻辑。

```typescript
import {
  ArgumentMetadata,
  Injectable,
  PipeTransform,
  HttpStatus,
  HttpException,
} from '@nestjs/common';
import { plainToInstance } from 'class-transformer';
import { validate } from 'class-validator';

@Injectable()
export class PipePipe implements PipeTransform {
  async transform(value: any, metadata: ArgumentMetadata) {
  	// value 就是 前端传过来的数据
    // metaData 就是元数据 通过 metatype 可以去实例化这个类
    const DTO = plainToInstance(metadata.metatype, value);

    const errors = await validate(DTO);

    // error长度为0时, 表示无错误信息
    if (errors.length > 0) {
      throw new HttpException(errors, HttpStatus.BAD_REQUEST);
    }

    return value;
  }
}

```



##### 2、全局校验
在`main.ts`中全局注册后, 在对应的`DTO`中设置校验逻辑均会校验。

```typescript
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';
import { Response } from './common/response';
import { HttpFilter } from './common/filter';

import { ValidationPipe } from '@nestjs/common';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);


  // 注册全局响应拦截器
  app.useGlobalInterceptors(new Response());

  // 注册全局异常拦截器
  app.useGlobalFilters(new HttpFilter());

  // 接口数据管道校验
  app.useGlobalPipes(new ValidationPipe());

  await app.listen(3000);
}
bootstrap();

```
