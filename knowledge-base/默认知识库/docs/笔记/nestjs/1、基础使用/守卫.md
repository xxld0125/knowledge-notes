<font style="color:rgb(77, 77, 77);">守卫有一个单独的责任。它们根据运行时出现的某些条件（例如权限，角色，访问控制列表等）来确定给定的请求是否由路由处理程序处理。这通常称为授权。在传统的 </font>`Express`<font style="color:rgb(77, 77, 77);"> 应用程序中，通常由中间件处理授权(以及认证)。中间件是身份验证的良好选择，因为诸如 </font>`token`<font style="color:rgb(77, 77, 77);"> 验证或添加属性到 </font>`request`<font style="color:rgb(77, 77, 77);"> 对象上与特定路由(及其元数据)没有强关联。</font>

<font style="color:rgb(77, 77, 77);">中间件不知道调用 </font>`next()`<font style="color:rgb(77, 77, 77);"> 函数后会执行哪个处理程序。另一方面，守卫可以访问 </font>`ExecutionContext`<font style="color:rgb(77, 77, 77);"> 实例，因此确切地知道接下来要执行什么。它们的设计与异常过滤器、管道和拦截器非常相似，目的是让您在请求/响应周期的正确位置插入处理逻辑，并以声明的方式进行插入。这有助于保持代码的简洁和声明性。</font>

<font style="color:rgb(77, 77, 77);"></font>

**<font style="color:rgb(77, 77, 77);">守卫在每个中间件之后执行，但在任何拦截器或管道之前执行。</font>**



#### 一、守卫实现
<font style="color:rgb(77, 77, 77);">守卫要求实现函数  给定参数</font>`<font style="color:rgb(77, 77, 77);">context</font>`**<font style="color:rgb(77, 77, 77);">执行上下文 </font>**<font style="color:rgb(77, 77, 77);">要求返回布尔值。</font>

```typescript
import { CanActivate, ExecutionContext, Injectable } from '@nestjs/common';
import { Observable } from 'rxjs';

@Injectable()
export class GuardGuard implements CanActivate {
  canActivate(
    context: ExecutionContext,
  ): boolean | Promise<boolean> | Observable<boolean> {
    console.log('进入 GuardGuard');
    return true;
  }
}

```



#### 二、Controller使用守卫
<font style="color:rgb(77, 77, 77);"> 使用</font>`<font style="color:rgb(77, 77, 77);">UseGuards</font>`<font style="color:rgb(77, 77, 77);">控制守卫。</font>

```typescript
import { Controller, UseGuards } from '@nestjs/common';
import { GuardService } from './guard.service';
import { GuardGuard } from './guard.guard';

@Controller('guard')
@UseGuards(GuardGuard)
export class GuardController {
  constructor(private readonly guardService: GuardService) {}
}

```



#### 三、全局守卫
注册全局守卫：`app.useGlobalGuards(new GuardGuard())`

```typescript
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';
import { Response } from './common/response';
import { HttpFilter } from './common/filter';

import { GuardGuard } from './guard/guard.guard';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  // 注册全局响应拦截器
  app.useGlobalInterceptors(new Response());

  // 注册全局异常拦截器
  app.useGlobalFilters(new HttpFilter());

  // 全局守卫
  app.useGlobalGuards(new GuardGuard());

  await app.listen(3000);
}
bootstrap();

```



#### 四、针对角色控制守卫
使用`SetMetadata`装饰器可以将<font style="color:rgb(77, 77, 77);">定制元数据附加到路由处理程序的能力。</font>

第一个参数为key，第二个参数自定义我们的例子是数组存放的权限。

```typescript
  @Get()
  @SetMetadata('role', ['admin'])
  findAll() {
    return this.guardService.findAll();
  }
```



在守卫`guard`中添加权限逻辑。

```typescript
import { CanActivate, ExecutionContext, Injectable } from '@nestjs/common';
import { Observable } from 'rxjs';
import { Reflector } from '@nestjs/core';
import type { Request } from 'express';

@Injectable()
export class GuardGuard implements CanActivate {
  constructor(private Reflector: Reflector) {}
  canActivate(
    context: ExecutionContext,
  ): boolean | Promise<boolean> | Observable<boolean> {
    // 访问链接路由参数携带role=admin时, 则表示有权限
    const admin = this.Reflector.get<string[]>('role', context.getHandler());
    const request = context.switchToHttp().getRequest<Request>();
    if (admin.includes(request.query.role as string)) {
      return true;
    } else {
      return false;
    }
  }
}

```



当权限不足的用户请求端点时，返回以下响应数据结构。

```json
{
  "statusCode": 403,
  "message": "Forbidden resource",
  "error": "Forbidden"
}
```
