实现新的 `NestedOmit<T, KeysToOmit>` 泛型，您将能够使用以下示例中所示的类型。

<font style="color:rgb(34, 34, 34);">此代码声明了一个名为 </font>`<font style="color:rgb(34, 34, 34);">SomeType</font>`<font style="color:rgb(34, 34, 34);"> 的类型，它具有嵌套属性的多级结构。 使用 </font>`<font style="color:rgb(34, 34, 34);">NestedOmit</font>`<font style="color:rgb(34, 34, 34);"> 泛型，传入类型，然后列出要省略的属性的键。 </font>

<font style="color:rgb(34, 34, 34);">请注意如何在第二个类型参数中使用点符号来标识要省略的键。 然后将结果类型存储在 Result 中。</font>

<font style="color:rgb(34, 34, 34);">构造此条件类型将使用 TypeScript 中可用的许多功能，例如，模板文字类型、泛型、条件类型和映射类型。</font>

```typescript
type SomeType = {
  a: {
    b: string,
    c: {
      d: number;
      e: string[]
    },
    f: number
  }
  g: number | string,
  h: {
    i: string,
    j: number,
  },
  k: {
    l: number,<F3>
  }
}

type Result = NestedOmit<SomeType, "a.b" | "a.c.e" | "h.i" | "k">;
```

  
	<font style="color:rgb(34, 34, 34);">要尝试这个泛型，首先创建一个名为 </font>`<font style="color:rgb(34, 34, 34);">NestedOmit</font>`<font style="color:rgb(34, 34, 34);"> 的泛型类型，它接受两个类型参数：</font>

```typescript
type NestedOmit<T extends Record<string, any>, KeysToOmit extends string>
```



<font style="color:rgb(34, 34, 34);">第一个类型参数称为 T，它必须是可分配给 </font>`<font style="color:rgb(34, 34, 34);">Record<string, any></font>`<font style="color:rgb(34, 34, 34);"> 类型的类型。 这将是您要从中省略属性的对象的类型。 </font>

<font style="color:rgb(34, 34, 34);">第二个类型参数叫做</font>`<font style="color:rgb(34, 34, 34);">KeysToOmit</font>`<font style="color:rgb(34, 34, 34);">，必须是字符串类型。 您将使用它来指定要从类型 T 中省略的键。</font>

<font style="color:rgb(34, 34, 34);">接下来，通过添加以下突出显示的代码来检查 </font>`<font style="color:rgb(34, 34, 34);">KeysToOmit</font>`<font style="color:rgb(34, 34, 34);"> 是否可分配给 </font>`<font style="color:rgb(34, 34, 34);">${infer KeyPart1}.${infer KeyPart2}</font>`<font style="color:rgb(34, 34, 34);"> 类型</font>

```typescript
type NestedOmit<T extends Record<string, any>, KeysToOmit extends string> = KeysToOmit extends `${infer KeyPart1}.${infer KeyPart2}`
```

<font style="color:rgb(34, 34, 34);">在这里，您使用模板文字字符串类型，同时，利用条件类型推断模板文字本身内部的其他两种类型。 </font>

<font style="color:rgb(34, 34, 34);">通过推断模板文字字符串类型的两个部分，您将字符串拆分为另外两个字符串。 第一部分将分配给 </font>`<font style="color:rgb(34, 34, 34);">KeyPart1</font>`<font style="color:rgb(34, 34, 34);"> 类型，并将包含第一个点之前的所有内容。 </font>

<font style="color:rgb(34, 34, 34);">第二部分将分配给 KeyPart2</font><font style="color:rgb(34, 34, 34);"> 类型，并将包含第一个点之后的所有内容。 如果您将“</font>`<font style="color:rgb(34, 34, 34);">a.b.c</font>`<font style="color:rgb(34, 34, 34);">”作为 </font>`<font style="color:rgb(34, 34, 34);">KeysToOmit</font>`<font style="color:rgb(34, 34, 34);"> 传递，则最初 </font>`<font style="color:rgb(34, 34, 34);">KeyPart1</font>`<font style="color:rgb(34, 34, 34);"> 将设置为确切的字符串类型“</font>`<font style="color:rgb(34, 34, 34);">a</font>`<font style="color:rgb(34, 34, 34);">”，而 </font>`<font style="color:rgb(34, 34, 34);">KeyPart2</font>`<font style="color:rgb(34, 34, 34);"> 将设置为“</font>`<font style="color:rgb(34, 34, 34);">b.c</font>`<font style="color:rgb(34, 34, 34);">”。</font>



<font style="color:rgb(34, 34, 34);">接下来将添加三元运算符来定义条件的第一个真分支：</font>

```typescript
type NestedOmit<T extends Record<string, any>, KeysToOmit extends string> =
  KeysToOmit extends `${infer KeyPart1}.${infer KeyPart2}`
    ?
      KeyPart1 extends keyof T
```

<font style="color:rgb(34, 34, 34);">这使用 </font>`<font style="color:rgb(34, 34, 34);">KeyPart1 extends keyof T</font>`<font style="color:rgb(34, 34, 34);">来检查 </font>`<font style="color:rgb(34, 34, 34);">KeyPart1</font>`<font style="color:rgb(34, 34, 34);"> 是否是给定类型 T 的有效属性。如果您确实有一个有效的键，请添加以下代码以使条件计算为两种类型之间的交集：</font>

```typescript
type NestedOmit<T extends Record<string, any>, KeysToOmit extends string> =
  KeysToOmit extends `${infer KeyPart1}.${infer KeyPart2}`
    ?
      KeyPart1 extends keyof T
      ?
        Omit<T, KeyPart1>
        & {
          [NewKeys in KeyPart1]: NestedOmit<T[NewKeys], KeyPart2>
        }
```

`<font style="color:rgb(34, 34, 34);">Omit<T, KeyPart1></font>`<font style="color:rgb(34, 34, 34);"> 是一种使用 </font>`<font style="color:rgb(34, 34, 34);">TypeScript</font>`<font style="color:rgb(34, 34, 34);"> 默认附带的 </font>`<font style="color:rgb(34, 34, 34);">Omit</font>`<font style="color:rgb(34, 34, 34);"> 助手构建的类型。 此时，</font>`<font style="color:rgb(34, 34, 34);">KeyPart1</font>`<font style="color:rgb(34, 34, 34);"> 不是点表示法：它将包含一个字段的确切名称，该字段包含您希望从原始类型中省略的嵌套字段。 因此，您可以安全地使用现有的实用程序类型。</font>

<font style="color:rgb(34, 34, 34);">您正在使用 </font>`<font style="color:rgb(34, 34, 34);">Omit</font>`<font style="color:rgb(34, 34, 34);"> 删除 </font>`<font style="color:rgb(34, 34, 34);">T[KeyPart1]</font>`<font style="color:rgb(34, 34, 34);"> 中的一些嵌套字段，为此，您必须重建 </font>`<font style="color:rgb(34, 34, 34);">T[KeyPart1]</font>`<font style="color:rgb(34, 34, 34);"> 的类型。 </font>

<font style="color:rgb(34, 34, 34);">为避免重建整个 </font>`<font style="color:rgb(34, 34, 34);">T</font>`<font style="color:rgb(34, 34, 34);">类型，您使用 </font>`<font style="color:rgb(34, 34, 34);">Omit</font>`<font style="color:rgb(34, 34, 34);"> 仅从 </font>`<font style="color:rgb(34, 34, 34);">T</font>`<font style="color:rgb(34, 34, 34);"> 中删除 </font>`<font style="color:rgb(34, 34, 34);">KeyPart1</font>`<font style="color:rgb(34, 34, 34);">，同时保留其他字段。 然后，您将在下一部分的类型中重建 </font>`<font style="color:rgb(34, 34, 34);">T[KeyPart1]</font>`<font style="color:rgb(34, 34, 34);">。</font>

<font style="color:rgb(34, 34, 34);"></font>

`<font style="color:rgb(34, 34, 34);">[KeyPart1 中的新键]</font>`<font style="color:rgb(34, 34, 34);">：</font>`<font style="color:rgb(34, 34, 34);">NestedOmit<T[NewKeys], KeyPart2></font>`<font style="color:rgb(34, 34, 34);"> 是一个映射类型，其中属性是可分配给 </font>`<font style="color:rgb(34, 34, 34);">KeyPart1</font>`<font style="color:rgb(34, 34, 34);"> 的属性，这意味着您刚刚从 </font>`<font style="color:rgb(34, 34, 34);">KeysToOmit</font>`<font style="color:rgb(34, 34, 34);"> 中提取的部分。 </font>

<font style="color:rgb(34, 34, 34);">这是您要删除的字段的父项。 如果您通过了 </font>`<font style="color:rgb(34, 34, 34);">a.b.c</font>`<font style="color:rgb(34, 34, 34);">，在第一次评估您的条件时，它将是“a”中的 </font>`<font style="color:rgb(34, 34, 34);">NewKeys</font>`<font style="color:rgb(34, 34, 34);">。 </font>

<font style="color:rgb(34, 34, 34);">然后将此属性的类型设置为递归调用 </font>`<font style="color:rgb(34, 34, 34);">NestedOmit</font>`<font style="color:rgb(34, 34, 34);"> 实用程序类型的结果，但现在使用</font>`<font style="color:rgb(34, 34, 34);"> T[NewKeys]</font>`<font style="color:rgb(34, 34, 34);"> 将此属性的类型作为第一个类型参数传递给 </font>`<font style="color:rgb(34, 34, 34);">T</font>`<font style="color:rgb(34, 34, 34);">，并作为第二个类型参数传递其余键以点表示法表示，在 </font>`<font style="color:rgb(34, 34, 34);">KeyPart2</font>`<font style="color:rgb(34, 34, 34);"> 中可用。</font>

<font style="color:rgb(34, 34, 34);">在内部条件的 </font>`<font style="color:rgb(34, 34, 34);">false</font>`<font style="color:rgb(34, 34, 34);"> 分支中，返回绑定到 </font>`<font style="color:rgb(34, 34, 34);">T</font>`<font style="color:rgb(34, 34, 34);"> 的当前类型，就好像 </font>`<font style="color:rgb(34, 34, 34);">KeyPart1</font>`<font style="color:rgb(34, 34, 34);"> 不是 </font>`<font style="color:rgb(34, 34, 34);">T</font>`<font style="color:rgb(34, 34, 34);"> 的有效键一样：</font>

```typescript

type NestedOmit<T extends Record<string, any>, KeysToOmit extends string> =
  KeysToOmit extends `${infer KeyPart1}.${infer KeyPart2}`
    ?
      KeyPart1 extends keyof T
      ?
        Omit<T, KeyPart1>
        & {
          [NewKeys in KeyPart1]: NestedOmit<T[NewKeys], KeyPart2>
        }
      : T
```

<font style="color:rgb(34, 34, 34);"></font>

<font style="color:rgb(34, 34, 34);">最后，在外部条件的 </font>`<font style="color:rgb(34, 34, 34);">false</font>`<font style="color:rgb(34, 34, 34);"> 分支中，使用现有的 </font>`<font style="color:rgb(34, 34, 34);">Omit</font>`<font style="color:rgb(34, 34, 34);"> 实用程序类型从 </font>`<font style="color:rgb(34, 34, 34);">Type</font>`<font style="color:rgb(34, 34, 34);"> 中省略 </font>`<font style="color:rgb(34, 34, 34);">KeysToOmit</font>`<font style="color:rgb(34, 34, 34);">：</font>

```typescript

type NestedOmit<T extends Record<string, any>, KeysToOmit extends string> =
  KeysToOmit extends `${infer KeyPart1}.${infer KeyPart2}`
    ?
      KeyPart1 extends keyof T
      ?
        Omit<T, KeyPart1>
        & {
          [NewKeys in KeyPart1]: NestedOmit<T[NewKeys], KeyPart2>
        }
      : T
    : Omit<T, KeysToOmit>;
```

<font style="color:rgb(34, 34, 34);"></font>

<font style="color:rgb(34, 34, 34);">如果条件 </font>`<font style="color:rgb(34, 34, 34);">KeysToOmit extends `${infer KeyPart1}.${infer KeyPart2}`</font>`<font style="color:rgb(34, 34, 34);"> 为假，这意味着 </font>`<font style="color:rgb(34, 34, 34);">KeysToOmit</font>`<font style="color:rgb(34, 34, 34);"> 没有使用点符号，因此，您可以使用现有的 </font>`<font style="color:rgb(34, 34, 34);">Omit</font>`<font style="color:rgb(34, 34, 34);"> 实用程序类型。</font>
