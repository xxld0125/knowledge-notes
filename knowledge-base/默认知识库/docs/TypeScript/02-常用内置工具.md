#### 一、索引类型查询操作符(keyof) & 索引访问操作符([])
##### 1、keyof
索引类型查询操作符keyof, 获取指定对象的公共属性名的联合。

```typescript
let person: Person = {
    name: 'Jarid',
    age: 35
};
let personProps: keyof Person; // 'name' | 'age'
```



##### 2、[]
下面使用的`T[K]`,就使用了 索引访问操作符，取到`interface T`中`K`索引对应的数据类型。

```typescript
function getProperty<T, K extends keyof T>(o: T, name: K): T[K] {
    return o[name]; // o[name] is of type T[K]
}

```



#### 二、Readonly
根据一个已知类型，生成一个新类型，新类型与已知类型拥有相同属性，但所有属性都为只读。

```typescript
interface Foo {
    name: string
    age: number
}
type Bar = Readonly<Foo>
// 相当于
type Bar = {
    readonly name: string
    readonly age: number
}
```



**源码**

生成一个新类型，T 中的 K 属性是只读的，K 属性是不可修改的。

```typescript
type Readonly<T> = {
    readonly [P in keyof T]: T[P];
};
```



#### 三、Partial
根据一个已知类型，生成一个新类型，新类型与已知类型拥有相同属性，但所有属性都为可选。

```typescript
interface Foo {
    name: string
    age: number
}
type Bar = Partial<Foo>
// 相当于
type Bar = {
    name?: string
    age?: number
}
```



**源码**

```typescript
type Partial<T> = {
    [P in keyof T]?: T[P];
};
```



#### 四、<font style="color:rgb(79, 79, 79);">Required</font>
根据一个已知类型，生成一个新类型，新类型与已知类型拥有相同属性，但所有属性都为必选。

```typescript
interface Foo {
    name: string
    age?: number
}
type Bar = Required<Foo>
// 相当于
type Bar = {
    name: string
    age: number
}
```



**源码**

```typescript
type Require<T> = {
    [p in keyof T]-?: T[P]
}
```



#### 五、Pick
根据一个已知类型T，和属性集K, 生成一个新类型，新类型相当于T 与 K的交集

```typescript
interface Foo {
    name: string;
    age?: number;
    gender: string;
}
type Bar = Pick<Foo, 'age' | 'gender'>
// 相当于
type Bar = {
    age?: number
    gender: string
}
```

****

**源码**

```typescript
type Pick<T, K extends keyof T> = {
    [P in K]: T[P];
};
```



#### 六、Exclude
根据一个已知类型T、U，如果T是U的子类型则返回`never`，不是则返回T中U不包含的属性集合。

```typescript
type A = number | string | boolean
type B = number | boolean

type Foo = Exclude<A, B>
// 相当于
type Foo = string
```



**源码**

```typescript
type Exclude<T, U> = T extends U ? never : T
```



#### 七、<font style="color:rgb(79, 79, 79);">Extract</font>
和 `Exclude` 相反

```typescript
type A = number | string | boolean
type B = number | boolean

type Foo = Extract<A, B>
// 相当于
type Foo = number | boolean
```



**源码**

```typescript
type Extract<T, U> = T extends U ? T : never
```



#### 八、Omit
根据一个已知类型T、属性集K，生成一个新类型，该类型拥有 T 中除了 K 属性以外的所有属性。

```typescript
type Foo = {
	name: string
	age: number
}

type Bar = Omit<Foo, 'age'>
// 相当于
type Bar = {
	name: string
}
```

****

**源码**

```typescript
type Omit<T, K extends keyof any> = Pick<T, Exclude<keyof T, K>> 
```



#### 九、NonNullable
从泛型T中排除`null` 和 `undefined`

```typescript
type t = NonNullable<'name' | undefined | null>;
// 相当于
type t = 'name'
```



**源码**

```typescript
type NonNullable<T> = T & {};
```



#### 十、Parameters
以元组的方式获得函数的入参类型

```typescript
type t = Parameters<(name: string) => any>;
// 相当于
type t = [string]

type t2 = Parameters<((name: string) => any) | ((age: number) => any)>;
// 相当于
type t2 = [string] | [number]
```



**源码**

```typescript
type Parameters<T extends (...args: any) => any> = T extends (...args: infer P) => any ? P : never;
```



#### 十一、<font style="color:rgb(79, 79, 79);">ConstructorParameters</font>
以元组的方式获得构造函数的入参类型

```typescript
type ConstructorParameters<T extends abstract new (...args: any) => any> = T extends abstract new (...args: infer P) => any ? P : never;
```



**源码**

```typescript
type t = ConstructorParameters<(new (name: string) => any) | (new (age: number) => any)>;
// type t = [string] | [number]
```



#### 十二、ReturnType
获得函数返回值的类型

```typescript
type t = ReturnType<(name: string) => string | number>
```



**源码**

```typescript
type ReturnType<T extends (...args: any) => any> = T extends (...args: any) => infer R ? R : any;
```



#### 十三、InstanceType
获得构造函数返回值的类型

```typescript
type t = InstanceType<new (name: string) => { name: string; age: number }>
```



**源码**

```typescript
type InstanceType<T extends abstract new (...args: any) => any> = T extends abstract new (...args: any) => infer R ? R : any;
```



#### 十四、in
可以遍历类型的索引进行对应操作。

比如我们可以写一个类型，在T的基础上，给T的每个属性加上联合类型`null`。

```typescript
type Nullable<T> = { 
  [P in keyof T]: T[P] | null;
}
```



#### 十五、Record
定义一个对象的 `key` 和 `value` 类型。

```typescript
interface PageInfo {
  title: string;
}

type Page = "home" | "about" | "contact";

const nav: Record<Page, PageInfo> = {
  about: { title: "about" },
  contact: { title: "contact" },
  home: { title: "home" },
};
```



**源码**

```typescript
type Record<K extends keyof any, T> = {
    [P in K]: T;
};
```



#### 十六、条件类型
条件类型是根据某些条件具有不同结果类型的泛型类型。 例如，看看下面的泛型类型 `IsStringType<T>`：

```typescript
type IsStringType<T> = T extends string ? true : false;
```

<font style="color:rgb(34, 34, 34);">在此代码中，您正在创建一个名为 </font>`<font style="color:rgb(34, 34, 34);">IsStringType</font>`<font style="color:rgb(34, 34, 34);"> 的新泛型类型，它接收单个类型参数 T。在您的类型定义中，您使用的语法看起来像使用 </font>`<font style="color:rgb(34, 34, 34);">JavaScript</font>`<font style="color:rgb(34, 34, 34);"> 中的三元运算符的条件表达式：</font>`<font style="color:rgb(34, 34, 34);">T extends string ？ 真假</font>`<font style="color:rgb(34, 34, 34);">。 </font>

<font style="color:rgb(34, 34, 34);">此条件表达式正在检查类型 T 是否扩展了类型字符串。 如果是，则结果类型将是完全正确的类型； 否则，它将被设置为 </font>`<font style="color:rgb(34, 34, 34);">false</font>`<font style="color:rgb(34, 34, 34);"> 类型。</font>

<font style="color:rgb(34, 34, 34);"></font>

**<font style="color:rgb(34, 34, 34);">注意</font>**<font style="color:rgb(34, 34, 34);">：此条件表达式是在编译期间求值的。 </font>`<font style="color:rgb(34, 34, 34);">TypeScript</font>`<font style="color:rgb(34, 34, 34);"> 仅适用于类型，因此请确保始终将类型声明中的标识符读取为类型，而不是值。 在此代码中，您使用每个布尔值的确切类型，true 和 false。</font>



#### 十七、infer
<font style="color:rgb(34, 34, 34);">条件类型的一个有用特性是它允许您使用特殊关键字 </font>`<font style="color:rgb(34, 34, 34);">infer</font>`<font style="color:rgb(34, 34, 34);"> 在 </font>`<font style="color:rgb(34, 34, 34);">extends</font>`<font style="color:rgb(34, 34, 34);"> 子句中推断类型信息。 然后可以在条件的真实分支中使用这种新类型。 此功能的一种可能用法是检索任何函数类型的返回类型。</font>

<font style="color:rgb(34, 34, 34);">编写以下 </font>`<font style="color:rgb(34, 34, 34);">GetReturnType</font>`<font style="color:rgb(34, 34, 34);"> 类型来说明这一点：</font>

```typescript
type GetReturnType<T> = T extends (...args: any[]) => infer U ? U : never;
```

<font style="color:rgb(34, 34, 34);">在此代码中，您将创建一个新的泛型类型，它是一个名为 </font>`<font style="color:rgb(34, 34, 34);">GetReturnType</font>`<font style="color:rgb(34, 34, 34);"> 的条件类型。 此泛型类型接受单个类型参数 T。</font>

<font style="color:rgb(34, 34, 34);">在类型声明本身内部，您正在检查类型 T 是否扩展了与函数签名匹配的类型，该函数签名接受可变数量的参数（包括零），然后您推断返回 该函数的类型创建一个新类型 U，可在条件的真实分支内使用。 </font>

<font style="color:rgb(34, 34, 34);">U 的类型将绑定到传递函数的返回值的类型。 如果传递的类型 T 不是函数，则代码将返回 never 类型。</font>

```typescript
type GetReturnType<T> = T extends (...args: any[]) => infer U ? U : never;

function someFunction() {
  return true;
}

type ReturnTypeOfSomeFunction = GetReturnType<typeof someFunction>;
```

<font style="color:rgb(34, 34, 34);">在此代码中，您将创建一个名为 </font>`<font style="color:rgb(34, 34, 34);">someFunction</font>`<font style="color:rgb(34, 34, 34);"> 的函数，该函数返回 </font>`<font style="color:rgb(34, 34, 34);">true</font>`<font style="color:rgb(34, 34, 34);">。 然后使用 </font>`<font style="color:rgb(34, 34, 34);">typeof</font>`<font style="color:rgb(34, 34, 34);"> 运算符将此函数的类型传递给 </font>`<font style="color:rgb(34, 34, 34);">GetReturnType</font>`<font style="color:rgb(34, 34, 34);"> 泛型，并将结果类型存储在 </font>`<font style="color:rgb(34, 34, 34);">ReturnTypeOfSomeFunction</font>`<font style="color:rgb(34, 34, 34);"> 类型中。</font>

<font style="color:rgb(34, 34, 34);">由于 </font>`<font style="color:rgb(34, 34, 34);">someFunction</font>`<font style="color:rgb(34, 34, 34);"> 变量的类型是函数，因此条件类型将评估条件的真实分支。 这将返回类型 U 作为结果。 </font>

<font style="color:rgb(34, 34, 34);">类型 U 是从函数的返回类型推断出来的，在本例中是布尔值。 如果检查</font>`<font style="color:rgb(34, 34, 34);">ReturnTypeOfSomeFunction</font>`<font style="color:rgb(34, 34, 34);"> 的类型，您会发现它已正确设置为布尔类型。</font>
