

日常开发中使用 `whistle + SwitchyOmega` 进行日常开发中的代理服务。

<font style="color:rgb(23, 43, 77);"></font>

whistle 是一款前端代理神器，需要依赖 node 环境，网页式操作使用，不占内存，支持各种形式的代理，个人感觉比 fiddle 好用

SwitchyOmega 是 chrome 的插件，用来代理 chrome 网页请求，配合 whistle 可以实现很多场景的代理

### 一、安装
#### 1、安装whistle
whistle依赖 node 环境，需要先安装node后，在安装whistle。

node安装后通过npm安装whistle

```bash
npm install -g whistle
```



#### 2、安装chrome插件-SwitchyOmega
1. 谷歌商店安装
2. github上安装最新版本：[https://github.com/FelisCatus/SwitchyOmega/releases](https://github.com/FelisCatus/SwitchyOmega/releases)



### 二、启动
```bash
# 启动
w2 start # 默认8899

# 指定端口启动
w2 start -p 8080

# 重启
w2 restart

#修改端口
w2 restart -o 8081

# 停止
w2 stop

# 查看其他命令
w2 help
```

### 五、常用的配置场景
#### 1、请求替换
1. 加载本地资源

```bash
test.wproxy.org/test file:///Users/xx/statics
```

> 请求test.wproxy.org/test的资源时，会尝试读取本地Users/xx/statics目录下的资源。
>
> eg：请求 test.wproxy.org/test/path/to 时，会尝试读取本地文件 /Users/xx/statics/path/to
>



2. 替换请求

```bash
# 加载本地的前端资源
https://kf-test.myyscm.com http://localhost:8080
```

> test.wproxy.org/test 及其子路径 test.wproxy.org/test/xxx 请求会用 https://ke.qq.com/xxx 响应头和内容替换
>



#### 2、查看js报错及页面`console.log`
指定域名后增加`log://`后,可以在network面板上看到，该域名页面的报错和`console.log`信息。

```bash
ke.qq.com log://
```

![](https://cdn.nlark.com/yuque/0/2022/png/25743026/1659680901106-f64632ef-7743-4c66-9e4a-ea14bd6f91ff.png)



#### 3、修改响应状态码
```bash
# 模拟响应 500（请求不会到后台服务）
test3.wproxy.org/path/to statusCode://500

# 修改响应状态码（请求会到后台服务）
test4.wproxy.org/path/to replaceStatus://400

# 302 重定向
test5.wproxy.org/path redirect://https://ke.qq.com/

# 301 重定向
test6.wproxy.org/path redirect://https://ke.qq.com/ replaceStatus://301
```





#### 4、修改接口信息
##### 1、修改请求信息
```bash
## 1.reqHeaders，修改请求头
## ua、referer协议都可以通过reqHeaders来完成
m.aliexpress.com reqHeaders://{req-headers}

{req-headers}
  {
    "X-Forwarded-For":" 188.146.171.71",
    "X-Real-IP":" 188.146.171.71",
    "X-Client-Scheme":"https",
    "referer":"https://vk.com/",
    "User-Agent":"Mozilla/5.0 (iPhone; CPU iPhone OS 9_1 like Mac OS X) AppleWebKit/601.1.46 (KHTML, like Gecko) Version/9.0 Mobile/13B143 Safari/601.1 AliApp(H/5)"
  }

```



##### 2、修改返回信息
```bash
## resHeaders，修改响应头
ae01.alicdn.com resHeaders://{res-cors}

## resReplace，替换响应的某些字符串
## 一般可用来快速验证某些功能
m.aliexpress.com resReplace://{res-replace}



#values
{res-cors}
  {
    "access-control-allow-origin":"*",
    "access-control-expose-headers":"Via"
  }

{res-replace}
  <meta name="screen-orientation" content="portrait">: <meta name="apple-itunes-app" content="app-id=436672029">
```



##### 3、其他修改的参数
+ **urlParams**

修改请求参数



+ **pathReplace**

<font style="color:rgb(51, 51, 51);">类似js字符串的replace方法，利用正则或字符串来匹配替换请求url的path部分</font>

<font style="color:rgb(51, 51, 51);"></font>

+ **<font style="color:rgb(51, 51, 51);">methods</font>**<font style="color:rgb(51, 51, 51);">:</font>

<font style="color:rgb(51, 51, 51);">修改请求方法</font>

<font style="color:rgb(51, 51, 51);"></font>

+ **<font style="color:rgb(51, 51, 51);">replaceStatus</font>**

<font style="color:rgb(51, 51, 51);">替换响应的状态码(状态码范围100~999)，这个与</font>[statusCode](http://wproxy.org/whistle/rules/statusCode.html)<font style="color:rgb(51, 51, 51);">的区别是，replaceStatus是请求响应后再修改状态码，而后者的请求不会发出去，设置完状态码直接返回</font>

<font style="color:rgb(51, 51, 51);"></font>

+ **<font style="color:rgb(51, 51, 51);">referer</font>**

<font style="color:rgb(51, 51, 51);">修改请求头的referer字段，有些服务器会校验请求头的referer字段，这个协议可以用来绕过这个检测或者测试后台的功能</font>

<font style="color:rgb(51, 51, 51);"></font>

+ **<font style="color:rgb(51, 51, 51);">auth</font>**

<font style="color:rgb(51, 51, 51);">修改请求头的</font>`<font style="color:rgb(51, 51, 51);background-color:rgb(247, 247, 247);">authorization</font>`<font style="color:rgb(51, 51, 51);">字段，这个字段是网页401弹出的输入框中输入用户名和密码的Base64编码</font>

<font style="color:rgb(51, 51, 51);"></font>

+ **<font style="color:rgb(51, 51, 51);">ua</font>**

<font style="color:rgb(51, 51, 51);">修改请求头的</font>`<font style="color:rgb(51, 51, 51);background-color:rgb(247, 247, 247);">user-agent</font>`<font style="color:rgb(51, 51, 51);">字段，可用于模拟各种机器访问</font>

<font style="color:rgb(51, 51, 51);"></font>

+ **<font style="color:rgb(51, 51, 51);">cache</font>**

<font style="color:rgb(51, 51, 51);">设置响应的缓存头</font>

<font style="color:rgb(51, 51, 51);"></font>

+ **<font style="color:rgb(51, 51, 51);">reqMerge</font>**

<font style="color:rgb(51, 51, 51);">修改表单请求的内容，包括普通的表单、上传表单及请求类型为JSON的内容(</font>`<font style="color:rgb(51, 51, 51);">Content-Type: application/json</font>`<font style="color:rgb(51, 51, 51);">)， 其功能是通过</font>`<font style="color:rgb(51, 51, 51);">reqMerge://jsonData</font>`<font style="color:rgb(51, 51, 51);">指定的JSON对象来覆盖请求表单或对应JSON对象字段，如果是GET请求的表单，则会直接修改URL参数。</font>

<font style="color:rgb(51, 51, 51);"></font>

+ **<font style="color:rgb(51, 51, 51);">resMerge</font>**

<font style="color:rgb(51, 51, 51);">修改响应类型为</font>`<font style="color:rgb(51, 51, 51);">application/json</font>`<font style="color:rgb(51, 51, 51);">或</font>`<font style="color:rgb(51, 51, 51);">text/javascript</font>`<font style="color:rgb(51, 51, 51);">的响应内容，其功能是通过</font>`<font style="color:rgb(51, 51, 51);">resMerge://jsonData</font>`<font style="color:rgb(51, 51, 51);">指定的JSON对象覆盖响应内容里面的JSON对象</font>

<font style="color:rgb(133, 133, 133);"></font>

其他配置见： [http://wproxy.org/whistle/rules/](http://wproxy.org/whistle/rules/)
