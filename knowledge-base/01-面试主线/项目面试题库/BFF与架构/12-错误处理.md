#### 一、框架基础错误类`<font style="color:rgb(28, 30, 33);">YunkeError</font>`
`<font style="color:rgb(28, 30, 33);">yunfly</font>`<font style="color:rgb(28, 30, 33);"> 内置了一些错误类，便于大家更好地快速定位问题以及统一错误信息。所有的内置 </font>

`<font style="color:rgb(28, 30, 33);">Error</font>`<font style="color:rgb(28, 30, 33);"> 类都是继承于 </font>`<font style="color:rgb(28, 30, 33);">YunkeError</font>`<font style="color:rgb(28, 30, 33);"> 。</font>

##### 1、框架内置错误结构
```typescript
interface YunkeError {
  // 错误名
  name: string
  // HTTP 状态码
  httpCode?: number
  // 自定义错误码
  code?: number | string
  // 错误信息
  message?: string
  // 错误详情
  // 因为有些信息不能返回给用户，但是需要记录到阿里云，故通过此属性保留
  messageDetail?: any
  // 错误堆栈信息
  stack?: string;
}
```



##### 2、name
<font style="color:rgb(28, 30, 33);">会自动读取当前 Error 类的类名，不需要配置，自动会有。</font>



##### 3、code
<font style="color:rgb(28, 30, 33);">code 的命名有三种规则：</font>

+ <font style="color:rgb(28, 30, 33);">规则 1：http 错误类</font><font style="color:rgb(28, 30, 33);">http 错误类 code 参考业界标准，例如</font><font style="color:rgb(28, 30, 33);"> </font><font style="color:rgb(28, 30, 33);">400、401、403、500、503</font><font style="color:rgb(28, 30, 33);"> </font><font style="color:rgb(28, 30, 33);">等。</font>
+ <font style="color:rgb(28, 30, 33);">规则 2：yunfly 框架及周边生态</font><font style="color:rgb(28, 30, 33);">从</font><font style="color:rgb(28, 30, 33);"> </font><font style="color:rgb(28, 30, 33);">10000</font><font style="color:rgb(28, 30, 33);"> </font><font style="color:rgb(28, 30, 33);">开头，每个应用占 100 个间隔，例如 redis 错误为 10100 - 10199，默认值是 10100</font>
+ <font style="color:rgb(28, 30, 33);">规则 3：rpc 调用错误</font><font style="color:rgb(28, 30, 33);">rpc 调用错误根据</font><font style="color:rgb(28, 30, 33);"> </font><font style="color:rgb(28, 30, 33);">golang</font><font style="color:rgb(28, 30, 33);"> </font><font style="color:rgb(28, 30, 33);">语言返回的结果确定，如果没有，则默认为</font><font style="color:rgb(28, 30, 33);"> </font><font style="color:rgb(28, 30, 33);">-1</font><font style="color:rgb(28, 30, 33);">。</font>
+ <font style="color:rgb(28, 30, 33);">规则 4：用户自定义错误严格上，用户自定义错误只要不和上述规则冲突即可，推荐从 40000 开始。</font>



##### 4、<font style="color:rgb(28, 30, 33);">httpCode</font>
<font style="color:rgb(28, 30, 33);">对于内置错误类而言，http 状态码规则如下：</font>

+ <font style="color:rgb(28, 30, 33);">HTTP 错误类，HTTP 状态码和 code 码是相同的，例如 401，http 状态码和错误码都为 401</font>
+ <font style="color:rgb(28, 30, 33);">对于 rpc 调用错误，统一 500</font>
+ <font style="color:rgb(28, 30, 33);">yunfly 框架及周边生态错误错误，导致服务器报错的，就是 500，其他视情况而定</font>
+ <font style="color:rgb(28, 30, 33);">用户自定义错误类，就由用户自己确定</font>

<font style="color:rgb(28, 30, 33);"></font>

##### <font style="color:rgb(28, 30, 33);">5、message</font>
<font style="color:rgb(28, 30, 33);">错误信息，可自定义。</font>

<font style="color:rgb(28, 30, 33);"></font>

##### <font style="color:rgb(28, 30, 33);">6、messageDetail</font>
<font style="color:rgb(28, 30, 33);">错误详情，例如在 rpc 调用时出现错误时，会返回很多的错误信息，这个信息实际是不需要给用户看的，我们可以通过赋值给 </font><font style="color:rgb(28, 30, 33);">messageDetail</font><font style="color:rgb(28, 30, 33);"> 属性来达到最终记录到阿里云日志的效果（框架会读这个字段）。</font>



##### 7、stack
<font style="color:rgb(28, 30, 33);">错误堆栈信息</font>

<font style="color:rgb(28, 30, 33);"></font>

#### <font style="color:rgb(28, 30, 33);">二、HTTP Error</font>
<font style="color:rgb(28, 30, 33);">继承于 </font><font style="color:rgb(28, 30, 33);">YunkeError</font><font style="color:rgb(28, 30, 33);"> 的错误类，用来表示 HTTP 请求相关错误。</font>

| **错误类** | **含义** | **HTTP 状态码** | **code 码** |
| --- | --- | --- | --- |
| BadRequestError | 参数错误 | 400 | 400 |
| UnauthorizedError | 用户未登录 | 401 | 401 |
| ForbiddenError | 用户无权限 | 403 | 403 |
| NotFoundError | 找不到相关资源 | 404 | 404 |
| MethodNotAllowedError | 方法不允许 | 405 | 405 |
| TooManyRequestsError | 过多请求 | 429 | 429 |
| InternalServerError | 代码不严谨导致的服务器错误 | 500 | 500 |
| BadGatewayError | 网关错误 | 502 | 502 |
| ServiceUnavailableError | 服务不可用 | 503 | 503 |
| GatewayTimeoutError | 网关超时 | 504 | 504 |




#### 三、Rpc Error
<font style="color:rgb(28, 30, 33);">继承于 </font><font style="color:rgb(28, 30, 33);">YunkeError</font><font style="color:rgb(28, 30, 33);"> 的错误类，用来表示 RPC 请求相关错误。</font>

| **错误类** | **含义** | **HTTP 状态码** | **code 码** |
| --- | --- | --- | --- |
| RpcError | RPC 错误 | 500 | code 由 go 返回的决定，默认值是 -1 |




#### 四、<font style="color:rgb(28, 30, 33);">yunfly 框架及周边生态 Error</font>
<font style="color:rgb(28, 30, 33);">继承于 </font><font style="color:rgb(28, 30, 33);">YunkeError</font><font style="color:rgb(28, 30, 33);"> 的错误类，用来表示 yunfly 框架以及周边生态（如：redis、apollo等）错误</font>

| ApolloError | Apollo 错误 | 500 | 10100 ~ 10199 |
| RedisError | Redis 错误 | 500 | 10200 ~ 10299 |
| EctdError | ECTD 错误 | 500 | 10300 ~ 10399 |
| MysqlError | Mysql 错误 | 500 | 10400 ~ 10499 |
| MongoDBError | MongoDB 错误 | 500 | 10500 ~ 10599 |
| GrpcError | Grpc 错误 | 500 | 10600 ~ 10699 |

#### 六、Error 类扩展
```typescript
import { YunkeError, YunkeErrorOptions } from '@yunke/errors'

// 继承 YunkeError
class CustomError extends YunkeError {
  // 其他几个信息也可以自定义
  constructor(options?: YunkeErrorOptions) {
    super(options)
    this.code = this.code ?? 40001
    this.message = this.message ?? '自定义错误信息'
  }
}
```





#### 七、兼容现业务场景
主要兼容前端对接口异常场景处理。

+ 鉴权失败：
    - errcode: 500
    - httpCode: 200
+ 接口异常：
    - errcode: !== 0
    - httpCode: 200
+ 接口正常：
    - errcode: 0
    - httpCode: 200
