### 配置化
##### 一、元数据配置
1. 请求接口

```javascript
{
      type: 'api', // 处理类型：接口请求
      dataKey: 'data', // 返回数据需要的字段(多层级的话可通过'.'获取);
      saveKey: 'taskDeadlineFlag', // 保存字段
      saveType: 'store', // 获取数据后的保存方式
      url: 'dailyservice-sso/engineer/test', // 接口请求地址
      params: [ // 接口请求参数
        {
          type: 'constant',
          key: 'func_code',
          value: 'task_deadline_flag'
        }
      ]
},
```

    - required: 设置表单初始必填校验状态  
配置同hidden

3. 自定义规则

```javascript
{
      type: 'condition',
      conditions: [
        {
          type: 'expression',
          expression: '===',
          leftDataSource: {
            type: 'formModel',
            dataKey: 'corp_name',
```

##### 二、dataService配置
服务暴露两个参数，context, $event；

+ context：当前元数据对应的core实例，可通过context上的方法获取元数据上相关数据;常用方法如下：
    - getFormModel： 获取当前元数据对应的formModel对象
    - getMetadataMap：获取以 _id 为键值缓存的元数据
    - getMetadataById：获取指定组件的元数据
    - getParentFormModelById：获取指定表单组件字段所挂载的 formModel 对象
    - getStore：获取store数据
    - getTempData： 获取临时数据挂载中心
    - getGrayPublish： 获取灰度开关列表
+ dataService：获取所有dataService方法



服务类型有区别的部分：

回调参数增加$event：事件的上下文数据

+ dataSource中的dataService类型
    - parseServiceName（调用API后对数据的处理）
    - dataService
+ condition
+ eventHandle



回调参数无$event：

+ initHandel
+ param



**使用示例**

```javascript
// metadata/index.js
export default {
    type: 'listPage',
    dataSource: [
        type: 'api',
        url: 'dailyservice/task-handle/list',
        {
            type: 'dataService',
            serviceName: 'getSearchParams'
        }
    ]
}


// dataService/index.js
function getParams({ context, $event, dataService }) {
    // 数据处理
}

export default {
    getParams,
}
```

```vue
    </config-form>

import comps from '.../index';

    components: {
        comps
    }
```
