1. 获取扩展点列表

主应用初始化扩展点，通过`appGroup`和`appName`，获取在贾维斯上配置的所有扩展点列表。

```javascript
import { extApp, extJs } from '@yunke/ext-react';
extApp({
  appGroup: 'kefu',
  appName: 'kefu-kffrontend-demo'
});
```

2. 是否匹配到扩展点

匹配扩展点逻辑：判断是否匹配到扩展点标识及满足匹配条件。



扩展点类型有：逻辑扩展点(函数扩展点)和外观扩展点(组件扩展点)。



逻辑扩展点匹配逻辑：

```javascript
import { extApp, extJs } from '@yunke/ext-react';

const currentOrgcode = new URL(window.location.href).searchParams.get('o');

function postUser() {
  console.error('主应用方法');
}

// 创建新函数
const extPostUser = extJs({
  name: 'user-form', // 扩展点标识
  checker: (conditions: any) => conditions.o === currentOrgcode, // 匹配条件
  originFn: postUser, // 原逻辑
  exportName: 'postUser',
});

async function handleClick() {
  // 调用新函数
  // ！！！注意，这里会变成异步的
  await extPostUser();
}
```



外观扩展点匹配逻辑：

```javascript
<ExtComponent
  name='user-form' // 扩展点标识
  exportName='ExtDemo'
  checker={(conditions) => conditions.o === currentOrgcode} // 匹配条件
>
    <div>主应用组件</div>
</ExtComponent>
```

6. 是否渲染错误

扩展点相关接口或者 SDK 内部出错，走原逻辑还是界面报错。

    - blockOnError: false 如果内部报错，总是会走原逻辑
    - blockOnError: true 如果内部报错，总是会阻塞原逻辑，并显示报错

```javascript
<ExtComponent
  name='user-form' // 扩展点标识
  exportName='ExtDemo'
  checker={(conditions) => conditions.o === currentOrgcode} // 匹配条件
  blockOnError={['retesting'].includes(currentOrgcode as string)}
/>
```



```javascript
const extPostUser = extJs({
  name: 'user-form', // 假定扩展点的标识叫 create-user
  checker: (conditions: any) => conditions.o === currentOrgcode, // 假定匹配条件为 1 时，调用扩展点
  originFn: postUser, // 原函数
  blockOnError={['retesting'].includes(currentOrgcode as string)}
  exportName: 'postUser',
});
```

8. 错误上报

配置errorHandler， 可将错误上报天眼。

```javascript
extJs({
  errorHandler: (error) => {
    // 假设使用天眼监控错误
    if(process.env.NODE_ENV !== 'development' &&  __myWebLogTracker__ && __myWebLogTracker__.report) {
      __myWebLogTracker__.report('扩展点错误', String(error))
      if (error instanceof Error) {
        __myWebLogTracker__.reportError(error)
      }
    } else {
      console.error(error)
    }
  }
});

```



9. 异常页面

errorFallback： 配置发生错误时，展示的站位信息

```javascript
<ExtComponent
  name='user-form'
  exportName='ExtDemo'
  checker={(conditions) => conditions.o === currentOrgcode} // 扩展点命中条件检测
  blockOnError={['retesting'].includes(currentOrgcode as string)}
  errorFallback='组件扩展点渲染失败'
/>
```

#### 四、其他
##### 1、单个扩展点应用多个租户
扩展点配置中匹配条件配置租户code， 用';'分开。

![](https://cdn.nlark.com/yuque/0/2023/png/25743026/1678957605525-26fbd715-ce86-4e3c-9123-b34ef076a001.png)



匹配逻辑在前端代码中, 可自定义。

```javascript
const currentOrgcode = new URL(window.location.href).searchParams.get('o');

const checker = (conditions: object) => {
  const orgCodeList = conditions.o?.split(';') || [];
  return orgCodeList.includes(currentOrgcode);
}

const extPostUser = extJs({
  name: 'user-form',
  checker: checker,
  originFn: postUser,
  blockOnError: false,
  exportName: 'postUser',
});
```
