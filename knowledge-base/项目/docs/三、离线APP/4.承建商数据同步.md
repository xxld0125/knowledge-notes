

APP中的需要的数据很多，对于根据不同的数据类型，APP的更新策略和时机并不一样。这样我们需要确认以下几点：

+ 更新什么数据
+ 什么时候更新
+ 怎么更新



#### 一、更新什么数据
基本上APP中使用的数据可以分为 基础数据 和 业务数据 。 业务数据就是用户在日常操作中产生或更新的数据。基础数据就是业务无关的数据。

根据上面对数据的定义，针对 基础数据 和 业务数据 也需要采取对应的更新数据。



#### 二、什么时候更新
##### 1.基础数据
APP中的基础数据不会因为用户的操作而产生数据变化，只需要在检测到数据发生变化时，下载新的数据即可。

那么基础数据需要再什么时候进行更新下载？

我们需要先区分基础数据是否与业务模块相关，将基础数据进行划分：

+ 与业务模块无关的基础数据通常是一些所有业务模块都遵守的标准，其更新时机一般是在每次进入APP首页时，会检查是否需要更新下载。
+ 与业务模块相关的基础数据，其更新时机一般是进入到对应业务模块的时候。



##### 2.业务数据
APP中的业务数据会根据用户的操作而产生变化，所以检测到数据需要更新时，要先上传数据，再更新下载数据。

更新时间一般是进入业务模块，或者业务模块刷新时。



#### 三、怎么更新
##### 1.基础数据
这个则是通过时间戳和版本号来判断，APP在进入首页会发起API请求去检查基础数据是否有变更，请求中会携带上次更新基础数据时记录的时间戳，服务端就可根据这个时间戳判断APP端的数据是否是最新的，如果需要更新则通知APP进行更新，APP更新完毕后又会保存新的时间戳。

时间戳可以解决数据更新的情况，但如果新的迭代里，基础数据表结构都发生变化了，那么在下发数据时就需要根据版本号进行控制是否下发新版数据结构了，否则当下发本地表没有的字段时，数据库的写操作就会异常导致程序不可用了。

##### 2.业务数据
业务数据的下载也依赖时间戳和版本号，相对于基础数据有什么不同的话，发起检查的API接口入参会添加业务模块对应的参数，用于给服务端判断。

##### 1.数据上传
首先需要明确上传的数据为用户在本地操作后产生或修改的数据，所以上传的数据为增量数据。那么我们如何本地表中哪些数据为增量数据呢？

在APP初始化时，我们需要创建一张增量表 kf_data_increment, 表结构如下


当用户进行操作时，除了处理对应的业务逻辑外，还需要再增量表中插入记录，比如当我们增加了一条派工单数据时，需要手动增加一条增量表记录。

##### 2.图片上传
上面介绍的数据增量上传是通过维护一张增量表判断的，图片上传与数据上传类似，也是在本地维护一张图片存储表(kf_work_images)。

表结构如下:

```sql
CREATE TABLE IF NOT EXISTS kf_work_images

  work_id              TEXT COLLATE NOCASE, -- 关联的工单id
  progress_id          TEXT COLLATE NOCASE, -- 关联的整改进展id
  proj_id              TEXT COLLATE NOCASE, -- 关联的项目id
  img_url              TEXT, 								-- 图片在线链接
  img_localpath        TEXT,								-- 图片的本地连接
  sort                 INTEGER DEFAULT 0,
  type                 TEXT									-- 图片分类

```



当我们在本地上传图片时，会将图片先存储到本地，得到一个本地的图片链接，然后将本地的图片链接插入图片表( kf_work_images )中，本地图片地址字段为img_localpath。



通过维护该表，我们可以通过判断图片有本地图片链接(img_localpath)且没有在线的图片链接(img_url)来确定该图片是否需要进行上传处理。

```javascript
// 根据租户code 和 项目id获取需要上传图片的sql
function getUploadImagesByProjectSql(tenantCode, projectId) {
  return "select images.id, images.img_localpath path,'" +
    tenantCode +
    "' || '/' || problem.proj_id upload_sub_dir " +
    'from kf_work_images images inner join kf_work problem on images.work_id = problem.id ' +
    "where images.proj_id = '" +
    projectId +
    "' and images.img_localpath is not null and images.img_localpath <> '' " +
    "and (images.img_url is null or images.img_url='')";
}
```

通过上面的sql， 可以获取到所有需要上传的图片数据。



上传完成之后，将所有已上传的图片数据插入上传后的在线图片链接(img_url), 避免下次上传时重复操作。

```javascript
function uploadByRules_prepareData (rules, config) {
  var dataMap = {};
  var data = [];
  var result = {
    data_map: dataMap,
    formatted_data: {
      endPoint: config.endPoint,
      federationAuthUrl: config.stsUrl,
      type: config.type,

    },
  };
  var tasks = $q.when();
  rules.forEach(function (rule) {
    tasks = tasks
      .then(function () {
        return localDB.query(rule.data_sql);
      })
      .then(function (rows) {
        rows.forEach(function (row) {
          // 值有效校验
          if (!row.id || !row.path || !row.upload_sub_dir) {
            throw new TypeError('uploadByRules_prepareData: row is invalid.' + JSON.stringify(row));
          }

          // 数据Map
          dataMap[row.id] = {
            id: row.id,
            fill_back_map: rule.fill_back_map,
          };

          data.push({
            bucket: config.bucketName,
            objectKey:
              config.parentDir +
              '/' +
              row.upload_sub_dir +
              '/' +
              row.id +
              (row.path.indexOf('.mp4') > -1 ? '.mp4' : '.jpg'),
            objectLocalPath: row.path,
            id: row.id,
          });
        });
      })
      .catch(err => {
        console.error(err);
      });
  });

  return tasks.then(function () { return result; })
};

function uploadByRules_execUpload (data, businessId, stsConfig) {
  var dataMap = data.data_map;
  var fillBackPromises = [];
  var item;
  let defer = $q.defer();
  if (!data.formatted_data.data.length) {
    console.log('uploadByRules_execUpload: 无文件需上传');
    defer.resolve('无文件需上传');
    return defer.promise;
  }

  pluginApi
    .uploadFilesToOSS(data.formatted_data, businessId, true)
    .then(
      null,
      function (error) {
        if (error.errCode == '10010') {
          // 图片不存在，删除记录，继续进行上传
          ProblemImagesRepository.deleteImageById(error.data.id);
          return {
            fileItem: error.data,
            fileTotal: error.total,
            fileDone: error.done,
          };
        } else {
          defer.reject(error);
        }
      },
      function (notify) {
        if (notify.fileItem) {
          item = dataMap[notify.fileItem.id];
          item.url = stsConfig.customDomain + '/' + notify.fileItem.objectKey;

          if (stsConfig.isAppendVersionToUrl) {
            item.url += '?' + new Date().getTime();
          }

          fillBackPromises.push(_private.uploadByRules_execUpload_fillBackUrl(item));
          defer.notify(notify);
        }
      }
    )
    .then(function () {
      return $q.all(fillBackPromises);
    })
    .then(function () {
      // 返回最后一个上传的对象
      defer.resolve(item);
    });
  return defer.promise;
};

  var updateObj = {};
```

###### 项目基础数据
项目数据的下载。项目数据又分为项目基础数据和项目业务数据。项目基础数据指的是与项目关联的，且不会受用户操作而导致数据变化。

项目基础数据下载跟之前的基础数据一样，也是通过对应的接口获取数据。但是写入本地数据库的方法有不同。

现将接口返回的数据接入到json文件中，用于插件读取，再通过插件将json文件中的内容写入本地数据库，具体实现见下：

```javascript
_self.createDataFile = function (fileName, content) {
  if (!content || !content.length) {
    return $q.resolve('');
  }

  // 如果是数组转成所要的json格式
  if (_.isArray(content)) {
    content = JSON.stringify({ result: content });
  }

  if (!_basePath) {
    _basePath = path.getOccasionalFilePath();
  }

  fileName = _basePath + fileName + new Date().getTime() + '.json';
  return pluginApi.createFile(fileName, content).then(function () {
    return fileName;

_private.updateData = function (now) {
  now = _.filter(now, function (val) {
    if (val) {
      return val;
    }
  });

  var promiseArray = [],
    promise = null;

  _.each(now, function (val) {
    promise = pluginApi.checkFileIsExists(val).then(function (isExists) {
      if (!isExists) {
        return $q.reject('数据文件不存在');
      }
    });

    promiseArray.push(promise);
  });
  return $q
    .all(promiseArray)
    .then(function () {
      return pluginApi
        .writeData(now)
        .then(
          function () {
            return $q.resolve();
          },
          function (error) {
            return $q.reject(error);
          }
        )
        .finally(function () {
          return _private.delete(now);
        });
    })
    .catch(function (error) {
      return $q.reject(error);
    });
};
```

```javascript
function downloadByPage (projId, totalPage) {
  // totalPage有什么作用？
  // 1. 有值，说明是增量下载，直接下载
  // 2. 无值，说明是全量下载，先下载第一页，获取总页数，再分页下载
  if (!totalPage) {
    return _private.downloadProblem(projId, 1).then(function (result) {
      if (!result) {
        return false;
      }
      totalPage = Math.ceil(result.kf_work_total / pageSize);
      // 没有数据直接返回成功
      return _private.updateProblemData(result, projId).then(function () {
        return totalPage;
      });
    });
  } else {
    var defer = $q.defer(),
      promise = $q.when();
    // 分页请求数据
    for (page = 1; page <= totalPage; page++) {
      (function (pageNow) {
        // 上一页请求成功才请求下一页数据
        promise = promise.then(function () {
          return _private.downloadProblem(projId, pageNow);
        });
      })(page);
    }
  }

  return defer.promise;
};
```

##### 4.图片下载
图片属于业务数据，再前面的业务数据下载完成后，图片表kf_work_images中已有下载的数据，但是此时表中的图片链接只有在线链接img_url，没有本地链接img_localpath。在下载过程中，需要将这个没有本地图片下载到本地。

```javascript
 function downloadImage (queue, update) {
  update = update || false;
  var defer = $q.defer();
  // 没有数据无需下载
  if (!queue.length) {
    defer.resolve();
  }

  var total = queue.length,
    done = 0;

  // 通过插件下载
  pluginApi.downloadFile(queue, true).then(
    function () {
      defer.resolve('download images success');
    },
    function (error) {
      if (!update) {
        defer.reject(error);
        return;
      }

      // 空间不足，返回错误, 其他错误跳过
      if (!_.isEmpty(error) && error.errMsg && error.errMsg.indexOf('空间不足') > -1) {
        Vue.prototype.$toast('糟糕，手机存储空间不足了');
        defer.reject(error);
      }
      if (done + 1 < total) {
        defer.notify(((done + 1) / total) * 100);
      } else {
        defer.resolve();
      }
    },
    function (notify) {
      if (!update) {
        defer.notify(notify);
        return;
      }
      var fileInfo = notify.fileInfo;
      if (!fileInfo) {
        defer.notify(notify);
        return;
      }
      total = notify.imageTotal;
      done = notify.imageDone;
      if (!notify.error) {
        // 下载完成后, 更新图片的本地连接
        _private.updateImagePath(fileInfo.url, fileInfo.path).then(function () {
          delete _imageDownloadQueue[fileInfo.url];
          if (done < total) {
            defer.notify({ queueId: notify.queueId, curr: done, total });
          } else {
            defer.resolve();
          }
        });
      }
    }
  );

  return defer.promise;
};

// 下载完成后, 更新图片的本地连接
 function updateImagePath (url, path) {
  var list = [],
    tableName = '',
    data = {};
  if (_imageQueue.problem_images[url]) {
    list = _imageQueue.problem_images[url];
    tableName = 'kf_work_images';

    data = { img_localpath: path };

    delete _imageQueue.problem_images[url];
  } else if (_imageQueue.room_type[url]) {
    list = _imageQueue.room_type[url];
    tableName = 'room_type';
    data = { diagram_localpath: path };
    delete _imageQueue.room_type[url];
  }

  return localDB
    .transaction(function (db) {
      _.each(list, function (id) {
        db.update(tableName, data, { id: id });
      });
    })
    .catch(function () {
      $log('更新图片本地路径失败');
      return $q.reject('更新图片本地路径失败');
    });
};
```
