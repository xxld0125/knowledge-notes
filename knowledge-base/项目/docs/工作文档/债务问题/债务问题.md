# <font style="color:rgb(23, 43, 77);">唠两句</font>
<font style="color:rgb(23, 43, 77);">客服的债务问题很多，原因很多：</font>

+ <font style="color:rgb(23, 43, 77);">有历史原因导致的技术被淘汰形成的债务</font>
    - <font style="color:rgb(23, 43, 77);">比如几个使用 jQuery 技术栈的老项目</font>
    - <font style="color:rgb(23, 43, 77);">比如承建商项目的 angularJs 技术栈的老项目</font>
+ <font style="color:rgb(23, 43, 77);">也有经手的人，尝试去解决这些债务问题产出的中间方案形成的另一债务</font>
    - <font style="color:rgb(23, 43, 77);">比如 vengineer 里的 v2.0 目录，七八年前的人重构并自研了一套路由机制的方案，但目前也被归为债务了</font>
    - <font style="color:rgb(23, 43, 77);">比如 v-service-online 里的  app-service-online 目录，也是七八年前的人，重构的一套，与 service-online 相比有什么差异，不清楚</font>
    - <font style="color:rgb(23, 43, 77);">比如 kfpack 构建工具，在老项目里去写 vue 代码的方案，虽然至今仍在使用，但其实也是个债务问题了</font>
+ <font style="color:rgb(23, 43, 77);">还有明明出发点是重构，结果时间太赶，需求太急，重构出了一堆太粗糙、太简陋的所谓新项目的债务</font>
    - <font style="color:rgb(23, 43, 77);">比如 paas 应用，急急忙忙重构，但很多细节没实现，导致不敢也无法全局推广</font>
    - <font style="color:rgb(23, 43, 77);">比如 400，完全交由来临时支援人重构的项目，基层等各种东西又自己搞了一套，很难全面推广到客服这种复杂的生产环境背景</font>
    - <font style="color:rgb(23, 43, 77);">比如 app-customer-service，出发点很好，但太多场景不支持，比如虽然把 appEngineerKit 离线代码在线化，但却只支持运行在 App 环境，仍旧导致无法全面推广</font>
+ <font style="color:rgb(23, 43, 77);">还有一些所谓的要提效进而自研封装的一套套工具，导致除了本人自己用起来爽外，对于其他人而已，都是债务、负担问题（没错，说的就是我封装的那一套套工具，对不住了，搞的时候确实没想这么多）</font>
    - <font style="color:rgb(23, 43, 77);">比如基于 vuex 封装的那一套动态注册、卸载、代码生成的工具</font>
    - <font style="color:rgb(23, 43, 77);">比如PC端的配置化表单（对不起，我知道除了我，这个对你们来说，确确实实是债务）</font>

<font style="color:rgb(23, 43, 77);">总之，说这么多，就是想让你们清楚：</font>

<font style="color:rgb(23, 43, 77);">债务不是随随便便来的，也许搞的人在当时，收益确实很多，对于当时来说，并不属于债务；</font>

<font style="color:rgb(23, 43, 77);">只是随着技术的发展和淘汰，经手人的各种各样理解，需求多变等这些东西慢慢形成了债务问题</font>

<font style="color:rgb(23, 43, 77);">所以，我希望的是，你们对待债务问题，在吐槽、嫌弃、质疑的同时，在你们打算推翻重来一套时，可以先多一点思考：</font>

+ <font style="color:rgb(23, 43, 77);">为什么会有这个债务问题出现？</font>
+ <font style="color:rgb(23, 43, 77);">他当时是不是为了解决什么问题而搞出的这个鬼东西？</font>
+ <font style="color:rgb(23, 43, 77);">你的这一次方案推翻重来一套，会不会把当时优先要解决的问题又带出来了？</font>

<font style="color:rgb(23, 43, 77);">我一直秉承着一个观念：</font>**<font style="color:rgb(23, 43, 77);">没有所谓的业内标准的最佳实践方案，只有性价比最高的实用方案</font>**

<font style="color:rgb(23, 43, 77);">有些时候，如果这个方案在投入产出的性价比上能给我带来很高的收益，还能解决我的诉求，那即时他存在一些缺点不足，只要不是太严重，其实都能接受</font>
